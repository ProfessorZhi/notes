# 已装插件总览

当前这个 vault 已安装并启用 4 个社区插件：

1. `Templater`
2. `GitHub Sync`
3. `Importer`

## 先看结论

如果只按你当前最值得上手的顺序排：

1. 先学 `Templater`
2. 再搞懂 `GitHub Sync`
3. `Importer` 按需使用
4. `Local REST API` 只有你明确要做自动化时再碰

原因很简单：

- `Templater` 直接改变你记笔记的效率
- `GitHub Sync` 影响同步和备份，配置不对会出问题
- `Importer` 是一次性迁移工具，不是日常核心插件
- `Local REST API` 很强，但也最容易带来安全和误操作风险

## 每个插件一句话

### Templater

它的本质不是“模板文件夹”，而是“在笔记里执行一套模板变量和脚本规则”。

适合：

- 日记模板
- 会议纪要模板
- 项目记录模板
- 自动插入日期、标题、光标位置

对应教程：
[01-Templater 上手教程.md](C:\Users\Administrator\Documents\Obsidian Vault\plugin\01-Templater 上手教程.md)

### GitHub Sync

它的本质不是“上传文件”，而是“本地自动提交，再和 GitHub 远端同步”。

适合：

- 桌面端轻量备份
- 单人使用
- 以电脑端为主的同步

不适合：

- 多端频繁双向编辑
- 想要稳定移动端 Git 工作流

对应教程：
[02-GitHub Sync 上手教程.md](C:\Users\Administrator\Documents\Obsidian Vault\plugin\02-GitHub Sync 上手教程.md)

### Importer

它是一次性迁移工具，用来把别的笔记系统导入到 Obsidian。

适合：

- 从 Notion、Evernote、OneNote、Apple Notes 迁移
- 导入 CSV、HTML、Textbundle 等外部内容

对应教程：
[03-Importer 上手教程.md](C:\Users\Administrator\Documents\Obsidian Vault\plugin\03-Importer 上手教程.md)

### Local REST API

它的本质是“把你的 Obsidian 变成一个本地 API 服务”。

适合：

- 外部脚本读写笔记
- 自动化工具接入
- 本地 AI / 脚本 / 工作流联动

不适合：

- 只是普通记笔记
- 没有自动化需求

对应教程：
[04-Local REST API 上手教程.md](C:\Users\Administrator\Documents\Obsidian Vault\plugin\04-Local REST API 上手教程.md)
