# Importer 上手教程

## 这个插件到底是干什么的

`Importer` 是迁移工具，不是日常写作插件。

它的作用只有一个：

- 把别的笔记系统内容导入 Obsidian

## 原理

它会读取外部格式的数据，然后尽量转换成：

- Markdown 文件
- Obsidian 可识别的附件
- 链接关系
- 文件夹结构

它的价值不在“功能多”，而在“减少迁移时的手工清洗成本”。

## 你什么时候才需要它

有以下情况再用：

- 从 Notion 迁到 Obsidian
- 从 Evernote 迁到 Obsidian
- 从 OneNote、Apple Notes、Bear、Roam 迁移
- 批量导入 CSV、HTML、Textbundle

如果你平时只是写 Obsidian 笔记，这个插件可以一直装着，但未必会天天用。

## 这个插件支持什么

你当前版本说明支持导入：

- Notion
- Evernote
- Apple Notes
- Microsoft OneNote
- Google Keep
- Bear
- Roam
- Textbundle
- CSV
- HTML

## 配置页面怎么理解

这个插件通常没有像 `Templater` 那样一大堆常驻设置。

它更像“导入向导”：

1. 你选择来源
2. 选择导入文件或目录
3. 选择导入目标位置
4. 执行导入

所以它不是“先长期配置好再用”的类型。

## 正确使用方式

### 第一步：先想清楚目标

导入前先定这三件事：

1. 导入到哪个文件夹
2. 附件放哪里
3. 原始层级要不要保留

否则导完后你会发现库结构很乱。

### 第二步：先小批量试一次

不要第一次就把整个历史库全导进来。

建议：

- 先拿一个小样本试导
- 看 Markdown 结果是否能接受
- 再决定是否全量导入

### 第三步：导入后再整理

导入工具解决的是“搬运”，不是“整理”。

你通常还需要后处理：

- 调整文件夹结构
- 修正标签
- 整理附件
- 修改命名方式

## 它适合什么，不适合什么

适合：

- 一次性迁移
- 半结构化数据导入
- 外部内容初步落地

不适合：

- 当作同步工具
- 当作日常自动整理工具
- 指望它完美还原原平台所有样式和行为

## 一句话建议

只有在“搬家”时才打开它。  
它是迁移工具，不是生产力核心插件。
