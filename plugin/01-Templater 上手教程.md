# Templater 上手教程

## 这个插件到底是干什么的

`Templater` 的本质不是“存几个模板文件”，而是：

- 让模板里能放变量
- 让变量在插入时动态计算
- 让模板能自动在新建文件时生效
- 让模板里能调用脚本和命令

所以它比 Obsidian 原生模板强的地方是“动态”。

例如：

- 自动插入今天日期
- 自动插入当前文件名
- 自动把光标跳到指定位置
- 新建某类文件时自动套模板

## 你该怎么理解它

### 原理

你写一个模板文件，里面放 `Templater` 命令。

当你插入模板时，插件会：

1. 读取模板文件
2. 解析里面的 `tp.xxx` 命令
3. 把结果替换成实际内容
4. 再写回当前笔记

所以：

- 模板文件是“输入”
- 解析后的文本是“输出”

## 建议你先这样配置

### 推荐目录

建议你在 vault 根目录新建一个文件夹：

`/Templates`

原因：

- 最简单
- 最容易记
- 路径最短
- 以后别的插件也容易配

### 推荐开局目标

先只做 3 件事：

1. 配好模板文件夹
2. 做一个日记模板
3. 学会“插入模板”

不要一开始就碰脚本、系统命令、复杂自动规则。

## 配置页面怎么理解

以下是最重要的设置项。

### `Template folder location`

作用：

- 指定“哪些文件夹里的文件可以被当作模板”

怎么配：

- 填 `/Templates`

推荐值：

- `/Templates`

### `Syntax highlighting on desktop`

作用：

- 在编辑器里给 Templater 语法加高亮

怎么配：

- 一般开启

推荐值：

- 开

### `Syntax highlighting on mobile`

作用：

- 在移动端编辑时给模板语法加高亮

问题：

- 官方文档提醒这个功能可能影响移动端实时预览

推荐值：

- 如果你移动端主要查看，关
- 如果你真要在移动端写模板，再试开

### `Automatic jump to cursor`

作用：

- 插入模板后，自动跳到模板里你用 `tp.file.cursor()` 标记的位置

推荐值：

- 开

这个功能很实用，尤其是会议纪要和日记模板。

### `Trigger Templater on new file creation`

作用：

- 新建文件时，自动执行模板规则

它的意义：

- 不是“插入已有模板”
- 而是“创建文件那一刻就自动套模板”

推荐值：

- 新手先关

为什么：

- 一旦你没配好规则，它会在你新建文件时自动改内容
- 容易让你分不清是自己写的还是模板自动写的

### `Template hotkeys`

作用：

- 给某个模板绑定快捷键

适合什么时候用：

- 你已经有固定高频模板
- 比如“会议纪要”“日报”“问题记录”

新手建议：

- 先不用

### `Folder Templates`

作用：

- 给某个文件夹下新建的文件自动套某个模板

原理：

- 你指定“文件夹 -> 模板”
- 在该文件夹或子文件夹里新建文件时自动用对应模板

适合：

- `Daily/` 统一套日记模板
- `Meetings/` 统一套会议模板
- `Projects/` 统一套项目模板

推荐：

- 等你先熟悉基础插入后再开

### `File Regex Templates`

作用：

- 用“文件路径正则规则”决定新文件该套哪个模板

原理：

- 不是按文件夹判断
- 而是按文件路径是否匹配某条正则

适合：

- 你的命名规则特别稳定
- 例如所有 `2026-xx-xx.md` 都套日记模板

推荐：

- 新手先别用

### `Startup templates`

作用：

- Obsidian 启动时执行一次模板

适合：

- 初始化一些脚本逻辑
- 挂接事件

推荐：

- 普通用户先别碰

### `User script functions`

作用：

- 从你自己的 JavaScript 文件里导入函数，给模板调用

适合：

- 你明确会写 JS

推荐：

- 先不配置

### `User system command functions`

作用：

- 让模板可以执行系统命令

风险：

- 这是最危险的功能之一
- 本质上是在模板里调用外部命令

推荐：

- 先不要开启

## 你现在最该怎么配

### 最小可用配置

1. 新建 `/Templates`
2. `Template folder location` 填 `/Templates`
3. 开启 `Syntax highlighting on desktop`
4. 开启 `Automatic jump to cursor`
5. 其他先保持默认或关闭

## 你可以先做的第一个模板

文件：

`/Templates/日记模板.md`

内容示例：

```md
# <% tp.date.now("YYYY-MM-DD dddd") %>

## 今日重点

- 

## 记录

<% tp.file.cursor() %>
```

效果：

- 标题自动变成今天日期
- 插入后光标直接跳到“记录”下面

## 最常用命令

### 日期

```md
<% tp.date.now("YYYY-MM-DD") %>
```

### 当前文件标题

```md
<% tp.file.title %>
```

### 光标位置

```md
<% tp.file.cursor() %>
```

### 创建时间

```md
<% tp.file.creation_date("YYYY-MM-DD HH:mm") %>
```

## 你真正该先学会的动作

1. 配模板目录
2. 新建一个模板文件
3. 用命令面板插入模板
4. 看懂 `tp.date.now` 和 `tp.file.cursor`

只要这 4 步通了，你就已经把 `Templater` 最有用的部分用起来了。

## 什么时候再学高级功能

等你出现下面任一需求时再继续：

- 想按文件夹自动套模板
- 想用文件名规则自动决定模板
- 想从 JS 里复用逻辑
- 想让模板执行系统命令

## 一句话建议

先把 `Templater` 当成“动态模板 + 光标跳转工具”来用。  
不要一开始把它当成脚本引擎。
