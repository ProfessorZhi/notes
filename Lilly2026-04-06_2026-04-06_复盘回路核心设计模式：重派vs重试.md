# 复盘回路核心设计模式：重派 vs 重试

## 来源

主管Bot复盘回路-MVP-2026-04-03.md

## 核心区别

结果不达标，但人找对了 → 返工（retry）：执行质量问题，要求同 agent 补具体缺口

结果不达标，但人找错了 → 改派（reroute）：路由问题，直接换 agent，不让原 agent 硬做

## 为什么这个区别重要

将"质量判断"和"路由判断"分开处理，避免两种错误混在一起导致：对正确 agent 做无效重试浪费一轮机会，或对错误的 agent 做返工结果原地打转。

## 最小实现要素

- 3 轮上限：每任务最多处理 3 次，防止无限循环和 token 成本失控
- 具体返工要求：不能说"重做"，必须指出具体缺口
- 改派不走重试流程：reroute 直接切换 agent 专项，不消耗重试轮次
- 阶梯链：绮雯→Sasha→Vivian 是合法链，说明 reroute 可以跨专项

## 一句话原则

主管判断两件事：结果够不够交付（质量验收），以及有没有找对人（路由判断）。两者独立，各有修正路径。

## 可迁移场景

- 多 agent 任务分派系统
- 带 human-in-the-loop 的工作流引擎
- 任何有专项路由的 orchestration 层

## 状态机视角

[指派] → [验收]
  → 通过 → [收口]
  → 质量不达标 → [返工]（同 agent，最多 3 次）
  → 路由不匹配 → [改派]（换专项 agent，轮次不重置）
