## 一、核心基础知识点（大模型八股）

大模型相关的八股与后端传统八股不同，没有后端那么成体系，主要是 **RAG（Retrieval Augmented Generation）**、**CoT（Chain of Thought）**、**ReAct** 这些核心概念。

### 1.1 基础知识架构

| 知识点 | 要求程度 | 面试频率 |
|--------|----------|----------|
| Transformer 架构 | 必须掌握 | 非常高 |
| GPT 架构 | 必须掌握 | 高 |
| 深度学习基础 | 了解 | 中 |

### 1.2 Transformer 原理（高频面试点）

面试中经常被问到 **自注意力机制（Self-Attention）**，需要能说出 **QKV 矩阵** 的三个计算过程：

```
自注意力机制计算流程：
1. Q（Query）：查询向量
2. K（Key）：键向量  
3. V（Value）：值向量

计算过程：
- 计算 Q 与 K 的点积 → 注意力权重
- 权重通过 Softmax 归一化
- 加权求和得到输出
```

### 1.3 AI Agent 调用参数

必须理解的参数：

- **Temperature（温度）**：控制输出的随机性
  - 值越低 → 输出越确定、越保守
  - 值越高 → 输出越随机、越有创意
- **Top-P**：控制候选词的概率阈值
- **Top-K**：限制每次采样的候选词数量

面试常见场景题：模型出现幻觉如何调参？

### 1.4 提示词工程（Prompt Engineering）

**提示词一般结构**：
- Role（角色设定）
- Instruction（任务指令）
- Constraints（约束条件）
- Examples（few-shot 示例）

**CoT（Chain of Thought）** 是八股常考点，需掌握：
- 概念定义
- 如何用提示词激活大模型的 CoT 能力
- 思维链的适用场景

### 1.5 工具调用

面试高频问题：**Function Calling 与 MCP 的区别**

## 二、Agent 系统架构

### 2.1 核心组成

```
用户输入 → Agent（大脑）
            ↓
         Plan 规划器
            ↓
      生成 To-Do List
            ↓
         执行器
            ↓
         结果反馈 → 循环
```

### 2.2 核心循环：ReAct 模式

**ReAct（Reasoning + Acting）** 是整个 Agent 开发的核心思想，也是 OpenAI Deep Research 等系统的底层逻辑：

```
ReAct 循环：
1. 调用工具
2. 获取结果
3. 大模型根据结果更新任务状态
4. 重复直到任务完成
```

## 四、RAG（检索增强生成）

### 4.1 核心流程

```
用户查询 → 切块分块 → Embedding（嵌入）→ 向量检索 → 生成回答
                          ↓
                     索引存储（向量数据库）
```

### 4.2 优化方向

| 优化方式 | 说明 |
|----------|------|
| 混合索引 | 结合稠密和稀疏向量 |
| Top-K 召回 | 召回相关度最高的结果 |
| 多路召回 | 同时使用多种召回策略 |
| 重排序（Rerank） | 对召回结果重新排序 |

**不同场景最佳方案不同，需视情况选择。**

## 五、Agent 相关概念

### 5.1 主要概念一览

| 概念 | 全称 | 说明 |
|------|------|------|
| **ReAct** | Reasoning + Acting | 推理+执行循环 |
| **CoT** | Chain of Thought | 思维链 |
| **ToT** | Tree of Thought | 思维树 |
| **MoA** | Mixture of Agents | 多 Agent 混合架构 |

### 5.2 MoA Agent（多 Agent 架构）

> 如果想做比较复杂的 Agent 系统，基本都会涉及 **MoA**（Multi-Agent）架构。

```
MoA 架构示例：
Agent A → Agent B → Agent C → 最终输出
   ↓          ↓          ↓
  子任务1    子任务2    子任务3
```

### 5.3 面试相关

- 腾讯三面曾问到：CoT 是什么、如何实现
- ReAct 是最近较火的 OpenAI Deep Research 的核心思想

## 六、上下文工程（Context Engineering）

### 6.1 核心问题

上下文工程是高频八股：面试官会问「你了解哪些上下文管理方案？」

### 6.2 推荐方案

#### 方案一：MemGPT

**论文**：基于操作系统思想的管理框架
- 借鉴操作系统的缓存、分页机制
- 核心思想：模拟操作系统管理内存的方式管理上下文

#### 方案二：海马体记忆框架（Mem）

> 仿造人体海马体的记忆框架

**层级结构**：
- 感觉记忆
- 短期记忆（工作记忆）
- 长期记忆

### 6.3 长期记忆存储

**向量数据库**是长期记忆的主流存储方式，原因：
- 避免上下文窗口爆炸（Context Window Overflow）
- 支持语义相似度检索

### 6.4 记忆管理核心

```
Prompt 编排 → 动态裁剪 → 记忆压缩 → 优先级排序
```

## 七、评测、安全与监控

> **这是区分「真正跑通完整 AI 链路」与「玩具项目」的关键。**

### 7.1 Agent 系统评测维度

| 类别 | 指标 |
|------|------|
| **RAG 质量** | 上下文召回率、答案准确度、答案相关性、上下文精度 |
| **系统鲁棒性** | 工具使用正确率、任务完成率 |
| **用户体验** | 人工介入率、响应延迟 |

### 7.2 评测方法

| 方法 | 说明 |
|------|------|
| **框架评测** | 使用 LangSmith 等工具自动评测 |
| **人工评测** | 人工评估生成质量 |

**常用工具**：LangSmith

### 7.3 监控的重要性

Agent 系统本质是 **黑盒**，需要大量监控链路来证明系统可用。这与后端系统监控的权重不同，后端开发监控是辅助，而 Agent 开发监控是核心。
