# Everything Claude Code 完全使用指南

## 项目简介

Everything Claude Code（简称 ECC）是 GitHub 上获得 116K star 的 Claude Code 配置项目，由黑客松获胜者花费近 10 个月打磨而成。

**核心定位**：如果说原版 Claude Code 是一个「聪明但没有约束的实习生」，ECC 就是给这个实习生配了一套完整的公司制度、操作手册、审查流程和监控系统。

## 安装配置

### 步骤一：添加市场并安装插件

### 步骤二：克隆仓库

```bash
# 克隆 ECC 仓库
git clone https://github.com/yknyang/Everything-Claude-Code.git
```

### 步骤三：复制配置文件

将仓库中的文件复制到本地电脑的 `.claude` 目录下。

## 六大核心模块架构

ECC 的模块协作逻辑：**slash 触发流程 → agents 干活 → skills 提供知识 → hooks 全程把关**

| 模块 | 数量 | 功能描述 |
|------|------|----------|
| **slash commands** | 60+ 个 | 斜杠命令，触发各种工作流 |
| **agents** | 28 个 | 专用 AI 子代理，各司其职 |
| **hooks** | 8 种类型 | 事件驱动的自动化脚本 |
| **skills** | ~125 个 | 领域知识库，自动加载规范 |
| **rules** | 全局 | 编码规范，对所有代理强制执行 |
| **mcp** | 外部集成 | 连接 GitHub、Superhuman 等服务 |

## 斜杠命令详解

斜杠命令是与 ECC 交互最直接的方式，是开发全流程的「技能栏」。

### plan - 规划命令（最被低估的功能）

**核心价值**：在动代码之前，让 AI 帮你把坑找出来。

**使用方式**：

```
/plan
```

**对比效果**：使用 `/plan` 命令相比不使用，能细化更多执行步骤，避免写到一半发现方向错误导致返工。

### TDD - 测试驱动开发

**核心价值**：颠覆传统的开发方法论，让 AI 先写测试再写实现。

**使用流程**：

```
/tdd
```

执行顺序：

1. 先写测试文件
2. 再写实现代码
3. 运行测试验证

### code review - 代码审查

**核心价值**：像机场安检一样，把本次修改的所有代码全部过一遍。

**触发方式**：

```
/code-review
```

- 检出安全问题
- 检出质量问题

### verify - 终极验收

**核心价值**：对标终极验收标准，要求代码必须全部通过以下三项：

- 构建（Build）
- 测试（Test）
- 安全扫描（Security Scan）

### 其他常用命令

| 命令 | 功能 |
|------|------|
| `/context` | 查看当前加载了多少字符，是否超出预算 |
| `/learn` | 从当前会话提取可复用模式 |
| `/instinct status` | 查看学习状态和置信度 |
| `/evolve` | 将本能合并升级为正式 skill 或 command |
| `/scale-create` | 从 Git 历史提取编码模式生成项目专属 skill |

## 子代理系统

ECC 内置 **28 个专用代理**，可理解为 AI 团队里的不同职能专家。

### 核心代理列表

| 代理名称 | 职责 |
|----------|------|
| `architect` | 架构设计 |
| `security-reviewer` | 安全审查 |
| `build-resolver` | 修复构建错误 |
| `integration-tester` | 集成测试 |

### 代理调用方式

**方式一：直接指定代理**

```
使用 security-reviewer 代理对 src/api/oss.ts 进行完整审查
```

**方式二：通过斜杠命令间接触发**

```
/code-review  # 触发 code-review 代理
```

大多数时候代理会自动根据上下文触发，无需手动调用。

## Hooks 钩子系统

Hooks 是 ECC 中最「隐形」但最稳定的功能，8 种事件类型支持事件驱动的自动化。

### 核心 Hook 类型

| Hook 类型 | 触发时机 | 功能示例 |
|-----------|----------|----------|
| `post-tool-use` | 工具使用后自动运行 | 编辑 .ts 文件后自动执行 Prettier 格式化 + TSC 类型检查 |
| `pre-tool-use` | 工具使用前拦截 | 写入文件前检查是否偷偷修改 linter 配置规避检查 |
| `stop` | 任务完成前 | 验收检查 |
| `session-start` | 新会话开始 | 加载记忆 |

### post-tool-use 示例

```typescript
// 编辑 .ts 文件后自动执行：
prettier --write {file}
tsc --noEmit {file}

// 编辑 .py 文件后自动执行：
black {file}
mypy {file}
```

### pre-tool-use 防作弊机制

防止 AI 为了让错误消失，偷偷修改配置文件关闭检查。这个 Hook 会：

1. 拦截任何文件写入操作
2. 检查是否修改了 linter 配置
3. 如有异常修改则阻止执行

**所有 Hooks 均预配置完成，安装 ECC 后自动生效，无需手动设置。**

## Skills 技能系统

### 概念说明

Skills 是领域知识库，Claude Code 执行任务时会**自动加载**相关 skill 作为参考。

### 内置规模

- 约 **125 个 skill**
- 覆盖：TypeScript、React、Python 等领域

### 工作机制

```
写 Django 代码 → 自动加载 Django 规范和安全检查清单
写 Go 代码 → 自动加载 Go 惯用模式和测试规范
```

**无需在提示词中写「请按 XX 规范」，Claude Code 自己知道。**

### 上下文管理

如果担心加载太多 skill 撑满上下文：

```bash
/context  # 查看当前加载了多少字符，是否超出预算
```

超出预算时可使用命令进行扩容。

## 基于本能的持续学习系统

ECC v2 版本引入的学习框架，Claude Code 可从会话中提取可复用模式，逐渐进化为专属工作流知识库。

### 学习工作流三步走

```
提取 → 积累 → 进化
```

### 第一步：提取模式

```bash
/learn
```

- 分析当前会话，找出值得复用的内容
- 给每个内容打一个**置信度分数**

### 第二步：查看学习状态

```bash
/instinct status
```

输出内容包括：

- 已学习的内容列表
- 每个内容的置信度分数
- **置信度越高，说明越确定这是你的编码习惯**

### 第三步：进化升级

```bash
/evolve
```

将积累的本能进行合并升级，变成正式的 **skill** 或 **command**，纳入个人工作流体系。

### 从 Git 历史挖经验

```bash
/scale-create
```

- 分析 Git 提交历史
- 自动提取过去的编码模式
- 生成项目专属的 skill

**价值**：将过去积累的好代码习惯系统化沉淀下来。
