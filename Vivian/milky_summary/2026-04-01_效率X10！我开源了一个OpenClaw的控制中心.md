# OpenClaw 控制中心：开源项目详解与使用指南

## 项目概述

**OpenClaw 控制中心**是一个将 OpenClaw 分散的运行信息集中展示的管理面板，解决多 Agent 协作时的信息可视化问题。

### 解决的核心痛点

当你认真使用 OpenClaw 跑多个 Agent、自动化任务、定时任务时，会遇到：

- 不知道谁在执行任务，谁卡住了，谁在等待处理
- 不知道今天消耗了多少 Token
- 不知道后面有多少定时任务正在排队
- Agent 的人设、记忆、任务文档等分散各处，无法统一查看

### 项目特点

- MIT 开源协议
- 支持二次定制
- 可按个人团队、工作流、审美继续修改

## 安装与接入

### 获取方式

两种方式获取项目：

1. **Git 克隆**
   ```bash
   git clone <仓库地址>
   ```

2. **下载压缩包**
   - 下载 zip 格式的压缩文件

### 接入步骤

**推荐方式：让 OpenClaw 自动安装**

不需要手动配置，只需在 OpenClaw 界面中完成以下配置：

| 配置项 | 说明 |
|--------|------|
| 环境 | 根据你的运行环境选择 |
| 网关 | 配置 API 网关 |
| 模型 | 选择你使用的模型 |
| Agent | 配置你的 Agent 列表 |

> 接入过程非一蹴而就，每个人的环境不同。OpenClaw 会逐步调整到最适合你的呈现状态。

## 功能详解

### 1. 总览页面（Dashboard）

**健康分机制**

当前健康分：100 分

采用**扣分制**，4 个评估维度：

| 扣分项 | 扣分规则 | 说明 |
|--------|----------|------|
| 待审阅队列 | 每项 -18 分 | 有任务卡住，需要人工审批才能继续 |
| 运行异常 | 每项 -18 分 | 异常停滞，无法正常执行 |
| 执行异常 | 每项 -18 分 | 无效执行，无产出但持续消耗资源 |
| 预算风险 | 每项 -18 分 | 预算达到当前限额的 80% |

总览页还会汇总左侧栏所有信息的状态。

---

### 2. Token 用量统计

**查看维度**

- **今日用量**：当天消耗的 Token
- **累计用量**：历史总消耗

**订阅用户关注重点**

对于订阅制用户，重要的是监控是否达到使用上限：

- 5 小时上限
- 一周上限

**任务消耗分析**

1. **任务类型统计**
   - 展示哪些类型的任务消耗了大部分 Token
   - 例如：定时任务、即时对话任务等

2. **具体任务统计**
   - 按单个任务维度查看消耗排名
   - 示例：X 上搜索情报的任务每 3 小时触发，消耗最多 Token

**优化价值**

通过用量统计可以判断：
- 哪些任务可以去掉
- 哪些任务需要保留
- 哪些任务可以优化/减轻消耗

---

### 3. Agent 状态管理

**Agent 列表页面**

查看所有 Agent 的当前状态：

- 是否在工作
- 最近的产出
- 排班情况

**Agent 详细信息**

每个 Agent 的配置信息：

| 信息项 | 说明 |
|--------|------|
| 模型 | 当前使用的模型 |
| 工作目录 | Agent 的工作路径 |
| 权限 | Agent 拥有的权限配置 |

---

### 4. 记忆管理

两种记忆类型：

**长期记忆（Long-term Memory）**

- 持续保留的上下文信息
- 例如：`memory.md` - Agent 的长期记忆文件

**对话记忆（Session Memory）**

- 当前对话线程中的上下文
- 命名格式：`session_` 开头

**支持的操作**

- 查看记忆内容
- 修改记忆内容

---

### 5. 任务文档管理

**支持查看和修改的文件**

- `agent.md` - 定义 Agent 的任务
- `heartbeat.md` - 心跳任务配置
- `todo.md` - 待办任务配置
- 其他 `.md` 格式的配置文件

---

### 6. 任务管理

**任务统计概览**

- 有效定时/心跳任务总数
- 今日待完成数量
- 今日已完成数量
- 明日继续任务数

**任务分类**

| 任务类型 | 说明 |
|----------|------|
| 定时任务 | 按设定时间周期触发 |
| 心跳任务 | 持续运行的监控任务 |

## 使用建议

### 优化方向

1. **基于用量数据进行任务优化**
   - 找出高消耗任务
   - 评估是否必要
   - 考虑优化方案

2. **关注健康分异常项**
   - 及时处理卡住的任务
   - 监控异常执行
   - 关注预算消耗

3. **定期维护 Agent 配置**
   - 更新人设文档
   - 调整记忆内容
   - 优化任务配置

### 扩展思路

项目可扩展的方向：

- 添加自定义信息面板
- 调整排版布局
- 对接更多数据源
- 定制团队专属视图

## 总结

OpenClaw 控制中心将原本分散的 Agent 运行信息、Token 消耗、任务状态等集中到一个可视化面板，特别适合以下场景：

- 运行多个 Agent 协同工作
- 使用大量定时任务/自动化任务
- 需要精细化管理 Token 消耗
- 想要告别 OpenClaw "黑盒"状态
