# AGTX 项目：多 CLI 智能体协作任务系统

## 项目概述

**AGTX** 是一个用于协调多个 AI 编程 CLI 工具（如 Claude Code、Claude Code X/Codex）协同完成任务的工具。

## 核心功能

### 1. 任务自动拆分与分发

```
用户输入任务
    ↓
AGTX 自动拆分任务
    ↓
分发到各个 CI 终端（Claude Code 实例）
    ↓
各终端并行执行子任务
```

### 2. 实时监控与对话

- 可以**实时查看每个 CI 终端正在执行的内容**
- 支持与各个智能体进行**对话交互**

### 3. 跨平台命令转换

- 不同 AI 工具的规则和命令格式各不相同
- AGTX 内置**网关（Gateway）**机制
- 将标准化命令**转换**为各 CI 终端适配的格式后再下发

### 4. MCP 协议通信

- 使用 **MCP（Model Context Protocol）** 进行智能体间的沟通与处理
- 各 CLI 工具加载 MCP 后可互相通信

### 5. TMUX 多面板协作

- 使用 **TMUX** 实现多终端协作界面
- 可同时观察多个终端的运行状态

## 当前存在的问题

| 问题 | 说明 |
|------|------|
| 过于重量级 | 架构复杂，不够轻量 |
| 无持久化 | 不能保存任务进度、Plan、文件系统 |
| 非对话群形式 | 缺乏类似飞书群/Open Crew 的使用体验 |

## 理想的解决方案

UP 主认为最佳的实现方式应该是**对话群模式**：

```
┌─────────────────────────────────────┐
│         协作群（类似飞书群）          │
├─────────────────────────────────────┤
│ Leader（用户）                       │
│  ↓ 可查看群内所有消息                 │
│  ↓ 可分配任务                         │
├─────────────────────────────────────┤
│ 子智能体 A（@提及才可被调用）          │
│ 子智能体 B（@提及才可被调用）          │
│ 子智能体 C（@提及才可被调用）          │
│ ...                                  │
└─────────────────────────────────────┘
```

### 核心特性

- **每个智能体都可以在群内发言**
- **Leader 和用户可查看群内所有消息**
- **子智能体必须被 @艾特 才会被调用**

## 现状分析

### 各工具的团队模式支持情况

| 工具 | 团队模式支持 | 状态 |
|------|-------------|------|
| Claude Code | ✅ 支持 | 目前唯一有 team mode 的工具 |
| Claude Code X/Codex | ❌ 不支持（最近才出子智能体模式） | 尚不支持团队协作 |

### 痛点

1. **Claude Code 存在的问题**：
   - 容易封号
   - 成本较高
   - 额度比 Codex 少

2. **Codex 存在的问题**：
   - 额度更多，但缺少 team mode
   - 难以实现多智能体协作

## 结论

目前还没有找到一个**轻量级、群聊式、支持多智能体协作**的工具。理想的项目形态应该是一个**对话群**，让用户和各预定义的智能体都能方便地沟通、调用和观察进度。
