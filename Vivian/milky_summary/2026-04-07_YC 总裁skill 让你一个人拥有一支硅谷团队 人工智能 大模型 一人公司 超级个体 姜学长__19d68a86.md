# gstack：一人拥有一个硅谷团队的生产力革命

## 什么是 gstack

gstack 是由硅谷顶级孵化器 YC（Y Combinator）总裁兼 CEO Gary Tan 亲自操刀开发的 AI 开发工具，在 GitHub 上狂揽 6 万多 star，是一个现象级的开源项目。

### 核心定位

gstack 的核心理念是：**把你手中单一的 AI 工具变成一整个建制完整的虚拟工程团队**。

### 核心数据

| 指标 | 数据 |
|------|------|
| 开发时间 | 60 天（兼职） |
| 代码量 | 60 万行生产级代码 |
| 内置专家角色 | 23 个 |
| 内置工具 | 8 个 |
| 可并行项目数 | 10-15 个 |
| 开源协议 | MIT |

---

## 标准开发流程

gstack 建立了一套雷打不动的标准化开发流程，每个环节都有专属的 AI 专家接手，上一步骤的输出会自动变成下一步骤的输入：

```
思考 → 计划 → 构建 → 审查 → 测试 → 发布 → 反思
```

---

## 四大核心维度

### 维度一：23 个专属虚拟专家，告别空白提示词

**解决的问题**：用户使用 AI 写代码时，常常盯着空白的提示词发呆，不知道该怎么下达指令。

**解决方案**：内置 23 个专家角色，通过简单的 markdown 或斜杠命令（`/`）调用。

**常用命令示例**：

| 命令 | 调用角色 | 功能 |
|------|----------|------|
| `/plan-ceo-review` | CEO / 创始人 | 重新思考产品的核心需求 |
| `/plan-engine-review` | 工程经理 | 敲定架构、数据流、测试矩阵 |
| `/office-hour` | 产品经理 | 通过直击灵魂的问题帮你理清需求 |

---

### 维度二：赋予 AI 真实的视力（Browser Agent）

**解决的问题**：传统 AI 只能阅读干巴巴的代码，无法像真人一样在真实环境中操作。

**核心能力**：

- 使用 `/qa` 和 `/bl` 命令可以直接打开真实的 Chrome 浏览器
- AI 可以像真人一样在浏览器中：
  - 点击按钮
  - 填写表单
  - 浏览页面
  - 主动寻找 bug

**自动化验证流程**：

1. 发现 bug
2. 自动改代码
3. 生成测试
4. 验证修复结果

**特殊场景处理**：如果遇到验证码或双重认证，AI 会将浏览器控制权限交还给用户手动操作，完成后 AI 可以继续接管工作。

---

### 维度三：真正的一人并行 10-15 个项目

**核心机制**：gstack 可以开启 10-15 个隔离的工作区，每个工作区独立运行不同的 AI 任务。

**并行工作示例**：

```
工作区 1：AI 构思点子
工作区 2：AI 审查代码
工作区 3：AI 在测试环境找 bug
工作区 4-9：AI 在不同分支修补漏洞
...
```

**用户角色**：你只需要像真正的负责人一样，做最后的审核即可。

**这就是 Gary Tan 能兼职写出 60 万行代码的真正秘诀。**

---

### 维度四：极度省心的自动化工作

#### 文档自动更新

使用 `document release` 命令，AI 会：

- 自动阅读你刚提交的代码
- 更新项目中所有的 README 文档
- 更新代办列表

#### 跨模型交叉审查

除了 Claude，gstack 还能引入 OpenAI 的 Codex 模型进行交叉审查：

```
Claude → 审查代码
Codex → 审查代码
```

不同公司的 AI 互相对抗检查同一个代码分支，极大提高了代码的可靠性。

## 总结

gstack 的本质不是普通的工具，而是一个**一人团队**。它通过：

1. **角色分工**：23 个虚拟专家覆盖产品、工程、测试各环节
2. **真实行动力**：AI 可直接操控浏览器执行真实操作
3. **并行能力**：10-15 个工作区同时运转
4. **自动化闭环**：从代码到文档到验证的全流程自动化

在这个超级个体的时代，一个人带着一个完整的 AI 团队做事，现在真的已经成为现实。
