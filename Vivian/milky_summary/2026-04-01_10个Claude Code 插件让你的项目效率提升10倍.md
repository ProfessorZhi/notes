# Claude Code 插件效率提升指南：10个必备工具详解

## 1. Supabase CLI（推荐）vs Supabase MCP

### 为什么选择 Supabase CLI
- **数据库管理**：支持标准 PostgreSQL 数据库和向量数据库（如 RAG）
- **身份验证**：处理用户登录、个人资料创建和数据隔离
- **免费额度**：提供慷慨的免费使用层级
- **开源**：基于开源工具构建，支持本地开发和自托管

### 安装方法
```bash
# 让 Claude Code 自动安装
"hey download and install the Supabase CLI tool"
```

### 重要原则：CLI 优于 MCP
- **CLI 工具**：专为 AI 编码代理设计，在终端中原生运行
- **MCP 工具**：增加额外开销，性能不如 CLI
- **选择建议**：当同时存在 CLI 和 MCP 版本时，始终选择 CLI

### 配置技巧
安装 CLI 工具后，需要添加相应的技能（skill）：
1. 检查官方 GitHub 或文档中的预构建技能
2. 在 Claude Code 插件市场搜索 "Cloudcode plugin"
3. 安装对应的技能以优化工具使用

## 2. Skill Creator Skill（最强大工具）

### 核心功能
- **创建自定义技能**：根据需求生成专用技能
- **修改现有技能**：优化和改进已有技能
- **性能测量**：提供数据驱动的技能评估

### 安装步骤
```bash
# 在 Claude Code 中执行
/plugin
# 搜索 "skill" 并安装 "skill-creator"
```

### 使用方法
```bash
# 方法1：直接调用
/skill-creator

# 方法2：自然语言
"hey Claude Code use the skill creator and I want to create a new skill"
```

### 价值亮点
- **AB 测试**：对比有技能和无技能的性能差异
- **版本对比**：测试技能修改前后的效果
- **数据驱动**：替代之前的"直觉判断"，提供具体性能数据

## 3. GSD Framework（Get Shit Done）

### 适用场景
- **新项目创建**：从零开始构建项目
- **规范化开发**：需要严格按阶段、按功能推进的项目

### 核心优势
- **规格驱动开发**：提供更多开发护栏
- **上下文管理**：
  - 防止上下文窗口腐败
  - 每个新阶段使用全新上下文窗口
  - 使用子代理避免后期输出质量下降

### 安装使用
```bash
# 在 GitHub 搜索 "GSD" 或 "get shit done"
# 执行安装命令后使用：
/GSD new project
```

### 工作流程
启动后会进入"超级计划模式"，详细确定项目需求和规划

## 4. Notebook LM Pi（研究工作流核心）

### 功能特色
- **连接 Notebook LM 和 Claude Code**：无需官方 API
- **全功能支持**：Notebook LM 的所有功能都可通过终端使用
- **免费价值**：利用 Notebook LM 的免费服务

### 支持的输出类型
- 视频、信息图表、幻灯片
- 闪卡、播客
- 研究分析和各类交付物

### 安装配置
```bash
# 1. 前往 notebook-lm-pi GitHub 仓库
# 2. 复制安装命令到终端
# 3. 安装依赖后登录
notebook-lm  # 会打开浏览器登录

# 4. 安装技能
notebook-lm-skill install
```

### 使用方式
用自然语言与 Claude Code 交流，所有 Notebook LM 功能都可通过终端实现

## 5. Obsidian（个人助手场景）

### 适用场景
- **个人助手用途**：管理大量 Markdown 文件
- **文档关联**：查看文本文件之间的连接关系
- **知识管理**：处理不断积累的上下文信息

### 配置方法
```bash
# 1. 下载 Obsidian
# 2. 创建指定文件夹作为 vault
# 3. 在该文件夹中打开 Claude Code 终端
# 4. 告诉 Claude Code 遵循 Obsidian 约定创建 Markdown 文件
```

### 注意事项
- **不适合编程项目**：主要用于个人知识管理
- **无需额外安装**：不需要 CLI 或特殊技能

## 7. Playwright CLI（浏览器自动化）

### 功能范围
- **网页操作**：自动化任何浏览器操作
- **购物自动化**：如自动添加商品到购物车并购买
- **UI 测试**：测试用户界面设计和交互

### 技术背景
- **开源工具**：Microsoft 开发的流行浏览器自动化工具
- **广泛应用**：在浏览器自动化领域使用广泛

### 安装步骤
```bash
# 1. 前往 Playwright CLI GitHub
# 2. 复制安装命令到终端
# 3. 安装技能
playwright-cli install --skills
```

### 应用场景
任何需要在浏览器中执行的操作都可以通过 Playwright 自动化完成

## 工具选择和使用建议

### 优先级原则
1. **CLI 优于 MCP**：性能更好，集成更自然
2. **添加对应技能**：安装 CLI 后记得安装相应的技能
3. **检查官方文档**：寻找预构建的技能和最佳实践

### 效率提升策略
- **组合使用**：不同工具配合使用效果更佳
- **场景匹配**：根据具体需求选择合适的工具
- **持续学习**：工具生态快速发展，保持更新

### 注意事项
- 工具生态发展迅速，即使专业人士也需要持续关注
- 选择工具时考虑实际需求，避免工具过载
- 优先使用官方推荐或社区验证的工具组合
