# 四种方法把工作 Skill 化

## 什么是 Skill

Skill 是封装好的 AI 工作模板，包含完整的上下文、指令和输出格式。不同于每次都重新写 prompt，Skill 让你调用一次就能产出标准化的交付物。

核心价值：

- **一致性**：每次输出符合同一标准
- **复用性**：不用重复描述背景和格式要求
- **迭代性**：使用过程中持续优化

---

## 四种制作 Skill 的方法

### 1. 反推法：从交付物入手

适合有明确成果标准的工作。

**操作步骤：**

1. 找到你最满意的一份交付物（PRD、周报、分析报告、短视频脚本等）
2. 将其发给 AI（Claude / ChatGPT 等）
3. 输入指令：

```
按这个标准来。我每次提供你 xxx 时，一起讨论，然后输出和上面类似的内容，制作一个 Skill。
```

4. AI 会提炼出这份交付物的结构、语气、必填字段，封装成 Skill

**案例：** 把写过的最好的 3 份 PRD 发给 Claude → 让它提炼结构 → 封装成「需求文档写作 Skill」→ 之后每次写 PRD 不从零描述格式

**适用场景：** 格式固定、质量标准明确的重复性交付

---

### 2. 流程法：从工作步骤入手

适合步骤固定、流程化的重复性工作。

**核心原则：** 先把每一步的输入输出定义清楚，再串成完整 Skill

**操作步骤：**

1. 拆解工作流中每个步骤
2. 为每个步骤制作独立的 Skill
3. 每个 Skill 跑顺后，合并成一个完整 Skill

**案例：** 播客处理工作流

```
拉更新 → 选播客 → 提字幕 → 内容提炼 → 存飞书 → 出图片
```

先把每个步骤分别用 Skill 实现，最后合并成一个「播客处理 Skill」

**提示：** 可以先完成其中一步，再逐步串联其他步骤

**适用场景：** 步骤明确、流程固定的自动化任务

---

### 3. 归纳法：从聊天记录入手

适合不知道该做什么 Skill 的情况。

**操作步骤：**

1. 回看最近一个月与 AI 的聊天记录
2. 让 AI 分析「我反复在让它做的是什么」
3. 找出高频重复的操作
4. 将其封装成 Skill

**案例：** 分析聊天记录后发现一个自己没注意的模式——几乎每次写完文章都会说「帮我换个更抓人的标题」，出现了十几二十次，但从来没做成 Skill。发现后立即制作「爆款标题制作 Skill」，之后写文章时直接调用，不用再重复发送提示词。

**适用场景：** 发现无意识的重复行为，优化使用效率

---

### 4. 经验提炼法：从隐性知识入手

适合难以言传的个人经验和团队规范。

**核心思路：** 每个人工作中都有「只可意会」的东西，把它们显性化、规则化

**操作步骤：**

1. 建立个人知识库文档（如 Obsidian 中的 `写作风格.md`、`审稿标准.md`）
2. 定义检查清单维度（如六个维度的标准）
3. 定期更新这些文档
4. 将文档绑定到对应 Skill

**案例：** 写作 Skill 绑定「写作风格.md」和「审稿标准.md」→ AI 每次动手前先读文件，不用手动发送风格要求

**可提炼的内容类型：**

- 个人写作风格
- 审稿标准
- 公司代码规范
- 团队设计原则
- 审批流程中的隐性规则

**适用场景：** 积累个人经验、沉淀团队规范

---

## 提升 Skill 使用效率的方法

### 1. 云端文档管理常用 Skill

把高频使用的 Skill 内容记录在云文档（Notion/Obsidian 等），调用时引用：

```
每次完成文章都要根据我的审稿标准来，根据我的写作风格来
```

云端 MD 文档可以让 AI 严格遵守执行。

### 2. 做界面统一管理

当 Skill 数量多时（案例：写作类 Skill 有 30+ 个），做前端界面管理，快速选择对应的 Skill。

### 3. 自动化重流程

将不需要人工参与的流程自动化，例如播客处理完全自动化执行，不用手动触发。

### 4. 轻量化任务放手机处理

适合移动端处理的任务：

- RSS 更新推送
- 关注博主的最新内容推送到飞书 IM

需要浏览器查看效果的任务放电脑处理：

- 代码修改
- 原型制作

### 5. GitHub 统一管理 Skill

**优势：**

- 换电脑也能使用同样的 Skill
- 版本控制、可追溯

**推荐做法：** 创建 GitHub 仓库统一存储所有 Skill

### 6. 用软链接同步 Skill 到多工具

如果你同时使用 VS Code、Obsidian、Claude Desktop 等工具，用软链接让它们访问同一个 Skill 文件夹。

**Windows 操作示例：**

```powershell
# 创建软链接（相当于快捷方式）
mklink /D "目标路径\Skill文件夹" "源路径\Skill文件夹"
```

### 7. 用环境变量管理 API Key

**问题：** 把 API Key 直接写在代码里，上传到 GitHub 公开仓库会泄露

**解决方案：** 使用环境变量存储 API Key

```python
import os

api_key = os.environ.get('API_KEY')
# 而不是直接写：api_key = "sk-xxxxxx"
```

### 8. Skill 自迭代

在 Skill 中加入自迭代能力：

**方式一：手动迭代**

- 每次使用后记录问题
- 定期更新 Skill 文档

**方式二：自动迭代**

在 Skill 中内置自迭代机制：

- 交付物不满意时，让 AI 修复
- 让 AI 记录修复内容并更新 Skill
- 每次使用 Skill 都是一次迭代优化

---

## 总结

| 方法 | 起点 | 适合场景 |
|------|------|----------|
| 反推法 | 已有交付物 | 有明确质量标准的成果 |
| 流程法 | 工作步骤 | 流程固定的任务 |
| 归纳法 | 聊天记录 | 不知道该做什么 Skill |
| 经验提炼法 | 隐性知识 | 个人经验和团队规范 |

关键原则：**Skill 不是一次性创建的，而是在使用过程中持续迭代优化的工具**
