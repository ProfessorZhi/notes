# Java 后端开发知识框架（Onboard 版）

> 目标不是把所有课程顺着看完，而是先把 **整张知识地图** 建起来，再按最短主干去打通。
>
> 这份框架按你要的思路重组：  
> **JavaSE → Java 高级（JVM/JUC 等）→ JavaWeb → 数据层 → 中间件 → 工具/底层 → Spring 全家桶 → 分布式/微服务 → 项目实战**
>
> 原则：
> 1. 先抓骨架，不先追细节。
> 2. 先抓依赖链，不先刷项目。
> 3. 先知道每块解决什么问题，再决定学多少。
> 4. 项目是“接通框架”的手段，不是起点。

---

## 一、整张地图：Java 后端到底在解决什么

从第一性原理看，Java 后端开发本质上是在解决 6 件事：

1. **把业务规则写成可运行程序**  
   这对应 JavaSE、面向对象、集合、异常、IO、并发。

2. **把程序暴露成网络服务**  
   这对应 HTTP、JavaWeb、Servlet、MVC、Spring MVC、Spring Boot。

3. **把数据稳定存下来并高效读写**  
   这对应 MySQL、Redis、MongoDB、Elasticsearch、事务、索引、缓存。

4. **让多个系统可靠协作**  
   这对应 MQ、RPC、Dubbo、Spring Cloud、注册中心、配置中心、分布式事务。

5. **让工程能持续开发、部署、演进**  
   这对应 Git、Maven、Linux、Docker、日志、监控、测试、CI/CD。

6. **在规模变大时仍然可维护、可扩展、可观测**  
   这对应 JVM、JUC、性能优化、限流熔断、微服务治理、系统设计。

## 二、最短主干路径

这是最适合 onboard 的主路径。  
**主干 = 不学它，后面所有东西都会漂；学会它，后面能快速挂载。**

### Stage 0：工程起步层
**解决的问题**：我怎么把代码写出来、跑起来、管理起来、部署出去。

- IDEA 基础使用
- 命令行基础
- Git
- Maven
- Linux 基础
- Docker 基础

> 结论：这一层不是“知识点”，而是整个学习过程的工作台。

---

### Stage 1：JavaSE 主干
**解决的问题**：程序如何表示数据、组织逻辑、抽象对象、处理一批数据。

- 程序执行模型：JDK / JRE / JVM、编译与运行
- 数据表示：变量、类型、运算符、字符串、数组
- 控制流：分支、循环、方法
- 抽象与建模：类、对象、封装、继承、多态、接口
- 容器：ArrayList、Collection、List、Set、Map、泛型
- 常用机制：异常、File、IO、反射、注解、网络基础
- 基础并发：线程、线程池、锁、并发工具类

> 这一层真正要建立的是：  
> **程序 = 数据结构 + 控制流 + 抽象边界 + 状态管理**

---

### Stage 2：Java 高级层
**解决的问题**：程序在运行时到底怎么工作，为什么会慢，会卡，会 OOM，会并发出错。

- JVM：内存结构、类加载、GC、对象创建、垃圾回收器、调优基础
- JUC：线程安全、锁、CAS、volatile、AQS、线程池、并发容器
- 性能与稳定性：CPU / 内存 / IO 视角、死锁、上下文切换、热点问题定位

> 这一层不是为了炫技。  
> 它的价值在于：你终于能解释“为什么系统会这样”，而不是只会调用 API。

---

### Stage 3：JavaWeb 层
**解决的问题**：一个 Java 程序怎样变成浏览器/前端能访问的服务。

- 网络基础：HTTP、TCP/IP、请求/响应
- Web 基础：Servlet、Filter、Listener
- 状态管理：Cookie、Session、Token
- MVC 思想
- 前后端分离的基本协作方式
- Web 项目的分层组织

> 这一层的本质是：  
> **把“本地程序”变成“网络中的服务”**

---

### Stage 4：Spring 全家桶核心层
**解决的问题**：怎么用成熟框架把 Web、业务、数据访问这三层真正组织起来。

- Spring Core：IoC / DI、Bean 生命周期、AOP
- Spring MVC：请求映射、参数绑定、统一返回、异常处理
- MyBatis / MyBatis-Plus：ORM / SQL 映射、动态 SQL、分页、插件
- Spring Boot：自动配置、starter、配置管理、日志、Profile、Actuator
- Spring 整合：事务、校验、拦截器、统一异常、文件上传、接口文档
- Spring Security / 鉴权（建议作为 Boot 后的能力模块）
- Spring AI（可作为扩展方向，不是主干前置）

> 这一层的本质是：  
> **用框架把“能跑的代码”升级成“可维护的工程”**

---

### Stage 5：数据层
**解决的问题**：业务数据怎么存、怎么查、怎么快、怎么一致。

#### 5.1 MySQL（关系型主线）
- 数据模型设计
- SQL：增删改查、连接、聚合、子查询
- 索引与执行计划
- 事务、隔离级别、锁、MVCC
- 性能优化：慢查询、分页、索引失效、主从/读写分离（进阶）

#### 5.2 Redis（缓存与高频状态）
- 数据类型
- 缓存模式：旁路缓存、缓存击穿/穿透/雪崩
- 分布式锁
- GEO / BitMap / HyperLogLog / Stream（按业务选择）
- 持久化、主从、哨兵、集群（进阶）

#### 5.3 MongoDB（文档型）
- 文档模型
- 适合场景与不适合场景
- 查询、索引、聚合

#### 5.4 Elasticsearch（搜索与分析）
- 倒排索引
- 索引、文档、mapping
- DSL 查询
- 聚合分析
- 搜索与数据库分工

> 这一层的本质不是“多学几个数据库”，而是：  
> **知道什么数据应该放在哪里，以及为什么。**

---

### Stage 6：中间件层
**解决的问题**：服务之间如何解耦、异步、削峰、广播、可靠通信。

#### MQ 主线
- RabbitMQ：经典入门、工作队列、交换机、路由、确认机制、死信
- Kafka：高吞吐日志流、分区、副本、消费者组、顺序性与重平衡
- RocketMQ：事务消息、延迟消息、顺序消息、金融/电商常见模型

#### 协调与 RPC
- Zookeeper：注册、协调、分布式元数据
- Dubbo：服务注册、发现、RPC 调用、负载均衡、治理

> 这一层要抓的是“消息模型”和“服务通信模型”，不是背命令。

---

### Stage 7：分布式 / 微服务层
**解决的问题**：单体系统拆开以后，如何还能稳定协作。

- 服务拆分原则
- 注册中心 / 配置中心（Nacos 等）
- API Gateway
- OpenFeign / RPC 调用
- 负载均衡
- 熔断 / 限流 / 降级 / 重试
- 分布式事务（2PC / TCC / 最终一致性思路）
- 幂等、去重、补偿
- 链路追踪、日志聚合、监控告警
- 容器化部署、环境隔离、配置管理
- 搜索、缓存、MQ 在微服务中的位置

> 这一层的本质是：  
> **从“一个进程里管理复杂性”，升级到“多个服务之间管理复杂性”**

---

### Stage 8：项目实战层
**解决的问题**：把前面各层真正接通。

推荐顺序：

1. **Tlias 教学管理系统**
   - 作用：适合进入 JavaWeb / Spring Boot / 基础 CRUD 工程化
   - 重点：接口、表结构、分层、前后端联调、部署

2. **苍穹外卖**
   - 作用：适合从单体企业项目视角看完整业务流
   - 重点：Spring Boot + SSM + 权限 + 文件上传 + 订单流转 + 工程结构

3. **黑马点评**
   - 作用：适合把 Redis 真正打穿
   - 重点：缓存模型、分布式锁、秒杀、GEO、热点 key、异步任务

4. **SpringCloud / 黑马商城 微服务项目**
   - 作用：适合把微服务各件套串起来
   - 重点：注册中心、配置中心、网关、MQ、ES、Docker、分布式事务

> 项目不是越早越好。  
> **项目要在你已经有主干骨架时进入，否则你只是在搬代码。**

## 三、按模块重组后的完整知识框架

## 0. 工作台与环境层
### 目标
建立最低可用开发环境与工程工作流。

### 模块
- 代码编辑器 / IDEA
- JDK 安装与版本管理
- 命令行基础
- Git
- Maven
- Linux 基础
- Docker 基础
- Nginx 基础（建议放部署阶段）

### 必须掌握
- 能独立创建、导入、运行 Java / Maven / Spring Boot 项目
- 能看懂目录结构与依赖
- 能用 Git 完成常见协作流
- 能在 Linux 上跑 Java 服务
- 能用 Docker 跑数据库、中间件、应用服务

## 1. JavaSE：程序设计基本功
### 目标
建立“如何写程序”的底层心智模型。

### 子模块
1. 程序执行模型
2. 基础语法与数据表示
3. 控制流
4. 数组与方法
5. 面向对象
6. 字符串与常用类
7. 集合体系
8. 异常
9. File / IO
10. 网络编程
11. 反射 / 注解 / 动态代理
12. 基础多线程

### 关键问题
- 数据怎么表示
- 行为怎么抽象
- 对象怎么协作
- 一批数据怎么组织
- 失败怎么处理
- 程序怎么和外部世界通信

### 不该过度沉迷
- API 细枝末节
- 过多算法题

## 2. Java 高级：运行时与并发
### 目标
把“会写代码”升级成“能解释代码为什么这样运行”。

### 子模块
1. JVM 内存结构
2. 类加载机制
3. 对象创建与逃逸分析（理解层）
4. GC 基础与常见收集器
5. JMM、volatile、synchronized
6. CAS、AQS、锁体系
7. 线程池
8. 并发容器
9. 性能定位与调优基础

### 必须掌握
- 为什么会 OOM
- 为什么会死锁
- 为什么有可见性问题
- 为什么线程池参数不能乱配
- 为什么 Full GC 会拖垮服务

---

## 3. JavaWeb：网络服务入口
### 目标
把本地程序变成 Web 服务。

### 子模块
1. HTTP / TCP/IP 基础
2. Servlet 生命周期
3. 请求 / 响应模型
4. Filter / Listener
5. Cookie / Session / Token
6. MVC 思想
7. 前后端分离接口协作

### 必须掌握
- 一个请求是怎么进来、怎么被处理、怎么返回的

### 4.3 MyBatis / MyBatis-Plus
- SQL 映射

- 分页
- 插件 / 拦截器

### 4.4 Spring Boot
- 自动配置
- starter
- 配置文件

- 日志
- 监控

### 4.5 典型通用能力
- 事务
- 接口文档
- 文件上传

### 目标
构建多数据系统视角，而不是只会 CRUD。

### 5.1 MySQL
- 表设计
- SQL 基础与高级查询
- 索引
- 事务与隔离级别
- 锁
- MVCC
- 执行计划
- 优化方法

### 5.2 Redis
- 五大数据类型 + 扩展类型
- 缓存模型
- 持久化
- 高可用
- 分布式锁
- 秒杀 / 热点 / 幂等辅助

### 5.3 MongoDB
- 文档模型
- 索引
- 聚合

### 5.4 Elasticsearch
- 倒排索引
- mapping
- DSL
- 聚合

### 核心心智
- MySQL 管“强一致结构化主数据”

## 6. 中间件：解耦与异步协作
### 目标
让系统之间通信更可靠、削峰更自然、扩展更容易。

### 子模块
1. MQ 基础模型
2. RabbitMQ
3. Kafka
4. RocketMQ
5. Zookeeper
6. Dubbo

### 必须掌握

- 顺序、可靠、幂等到底怎么取舍
- 注册中心在分布式里扮演什么角色
- RPC 为什么需要治理

---

## 7. 分布式 / 微服务：跨服务治理
### 目标
把单体系统的复杂性，升级为跨服务复杂性的治理能力。

### 子模块
1. 服务拆分与边界
2. 注册中心
3. 配置中心
4. 网关
5. 远程调用
6. 负载均衡
7. 容错机制
8. 分布式事务
9. 幂等 / 去重 / 补偿
10. 可观测性
11. 容器化与部署

### 必须掌握
- 微服务不是为了炫架构，而是为了解决组织规模与系统复杂度问题
- 一旦拆服务，就会引入一致性、延迟、容错、部署复杂度
- 分布式问题本质上是“网络不可靠 + 状态分散”

---

## 8. 工程与底层：真实开发底座
### 目标

## 9. 计算机基础（建议并行补）

## 七、项目在这张地图里的正确位置

### Tlias
**位置**：JavaWeb / Spring Boot 初次接通项目  
**作用**：让你把接口、数据库、前后端联调、部署第一次跑通

### 苍穹外卖
**位置**：Spring Boot 单体企业项目  
**作用**：让你看到真实业务流和工程结构

### 黑马点评
**位置**：Redis / 高并发 / 场景型项目  
**作用**：把缓存、锁、秒杀、热点数据这些问题真正落地

### SpringCloud 微服务项目 / 黑马商城
**位置**：分布式 / 微服务综合项目  
**作用**：把服务注册、配置、网关、消息、搜索、部署接起来

> 一个项目只负责帮你接通一段主线。  
> 不要把任何单个项目误以为“学完整个 Java 后端”。

## 九、最后的结论

真正的 Java 后端知识体系，不是“课程列表”，而是一条主干链：

**写程序 → 做服务 → 管数据 → 提性能 → 做工程 → 解耦协作 → 治理分布式系统**

你可以把整张图压缩成一句话：

> **JavaSE 负责把程序写出来，JavaWeb 和 Spring 负责把程序变成服务，数据库和缓存负责把状态管理好，中间件和微服务负责把多个服务组织起来，工具与底层负责让整个系统能长期稳定运行。**
