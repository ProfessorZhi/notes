# Java 后端 Onboard 工作台

> 这不是资料清单，而是一张可推进、可复盘、可纠偏的主干作战图。
>
> 用法：
> 1. 只推进当前阶段，不并行乱学。
> 2. 每过一阶段，只检查“能不能解释 + 能不能做最小实践”。
> 3. 项目只拿来接通主线，不拿来代替理解。

---

## 0. 总目标

用尽可能短的时间，建立 Java 后端开发的完整知识地图，并打通一条最短可用主线：

**JavaSE → JavaWeb / SpringBoot → MySQL / Redis → 工程化 → 项目 → 微服务**

不是先学全，而是先具备：
- 能独立写、跑、调 Java 程序
- 能独立写基础 Web 接口
- 能把数据落到数据库
- 能理解缓存、事务、分层、依赖注入这些核心机制
- 能做一个单体后端项目
- 能再向微服务、高并发、分布式扩展

## 2. 当前最短主干路径

### Phase 0：工作台搭建
**目标**：先把开发环境和工程操作手感建立起来。

#### 只学这些
- IDEA 基本使用
- JDK 安装与版本切换
- Git 基本工作流
- Maven 基本结构与依赖管理
- Linux 常用命令
- Docker 基本概念与常用命令

#### 最小实践
- 用 IDEA 新建一个 Maven Java 项目并运行 `main`
- 提交到 Git 仓库，完成一次 commit
- 用 Docker 跑一个 MySQL 容器

#### 过关标准
- 你能解释：Maven 是干嘛的，Git 是干嘛的，Docker 是干嘛的
- 你能独立创建项目、导入依赖、运行程序、提交代码

#### 暂时别深挖
- Git 底层对象模型
- Maven 生命周期细枝末节
- Docker 镜像构建高级玩法

---

### Phase 1：JavaSE 主干
**目标**：建立“程序如何表示数据、组织逻辑、抽象对象”的心智模型。

#### 必学主干
1. 程序执行模型
   - JDK / JRE / JVM
   - 编译 vs 运行
2. 数据表示
   - 变量、类型、运算符、字符串、数组
3. 控制流
   - if / switch / for / while
4. 方法
   - 参数、返回值、复用、抽象边界
5. 面向对象
   - 类、对象、封装、继承、多态、接口
6. 集合
   - List / Set / Map
   - 泛型
7. 异常
8. File / IO
9. 基础多线程
10. 反射 / 注解 / 动态代理（先知道位置）

#### 这一层真正要抓住的 6 个问题
- 程序里的数据怎么表示？
- 逻辑怎么组织？
- 为什么要有方法？
- 为什么要有对象？
- 为什么要有集合？
- 异常、IO、线程分别在解决什么现实问题？

#### 最小实践
- 学生管理系统
- 图书管理系统
- 简单文件读写工具
- 一个最小生产者-消费者线程示例

#### 过关标准
- 你能不用背稿解释面向对象是干嘛的
- 你能说清 List、Set、Map 的差异和适用场景
- 你能自己写一个带增删改查的控制台小系统

#### 暂时别深挖
- 冷门 API
- 复杂反射细节
- 网络编程细节
- JUC 深水区

---

### Phase 2：JavaWeb 与 HTTP
**目标**：把“本地程序”升级成“网络里的服务”。

#### 必学主干
- HTTP 请求/响应模型
- URL / 方法 / 状态码 / Header / Body
- Servlet
- Filter / Listener
- Cookie / Session / Token
- MVC 思想
- 前后端分离接口协作方式

#### 这一层真正要抓住的 4 个问题
- 浏览器为什么能调你的 Java 程序？
- 请求是怎么进来的？
- 状态为什么会丢，需要 Session / Token？
- Controller、Service、DAO 为什么分层？

#### 最小实践
- 写一个最小登录接口
- 写一个最小 CRUD 接口
- 写一个过滤器记录请求日志

#### 过关标准
- 你能说清一次 HTTP 请求从浏览器到 Java 服务的大致链路
- 你能说清 Cookie / Session / Token 的差异
- 你能手写一个最小 Servlet 或至少看懂其职责

#### 暂时别深挖
- Servlet 容器底层源码
- 复杂前端技术栈

---

### Phase 3：Spring / SpringBoot 主干
**目标**：理解现代 Java 后端项目为什么这样组织。

#### 必学主干
1. Spring Core
   - IoC
   - DI
   - Bean
   - AOP 的作用
2. SpringMVC
   - 请求映射
   - 参数绑定
   - 返回结果
   - 统一异常处理
3. MyBatis / MyBatis-Plus
   - SQL 映射
   - Mapper
   - 基本 CRUD
4. SpringBoot
   - 自动配置
   - starter
   - application.yml
   - Profile
   - 日志

#### 这一层真正要抓住的 5 个问题
- 为什么需要 IoC，而不是手动 new？
- Controller / Service / Mapper 为什么分层？
- SpringBoot 为什么能“少配置就跑起来”？
- MyBatis 到底在帮你省什么？
- AOP 适合解决什么横切问题？

#### 最小实践
- 用 SpringBoot + MyBatis 写一个员工管理/部门管理系统
- 加入参数校验、统一返回体、全局异常处理
- 接 MySQL 完成基础 CRUD

#### 过关标准
- 你能解释依赖注入的价值
- 你能从 0 写出一个最小 SpringBoot CRUD 项目
- 你能看懂常见项目的分层目录结构

#### 暂时别深挖
- Spring 源码细节
- 自动配置源码大面积跟读
- AOP 底层代理实现细枝末节

---

### Phase 4：MySQL 与数据建模
**目标**：理解“业务数据如何被设计、存储、查询、更新”。

#### 必学主干
- 表、行、列、主键、外键
- 一对一 / 一对多 / 多对多
- 基础 SQL
- 连接查询、聚合、分页
- 索引
- 事务
- 隔离级别
- 锁 / MVCC（先到能解释问题）

#### 这一层真正要抓住的 5 个问题
- 业务对象怎么落成表结构？
- 为什么会需要索引？
- 为什么事务会存在？
- 为什么会有脏读、不可重复读、幻读？
- 分页为什么会慢？

#### 最小实践
- 给你的 SpringBoot 项目设计库表
- 完成用户、角色、订单/任务等基础表设计
- 写出常见联表查询
- 分析一条慢 SQL 的潜在原因

#### 过关标准
- 你能独立做中小型业务的表设计
- 你能说清索引的核心价值
- 你能说清事务是在兜什么底

#### 暂时别深挖
- 复杂分库分表
- MySQL 源码
- 过早钻复杂优化技巧

---

### Phase 5：Redis 与缓存模型
**目标**：理解“为什么数据库扛不住，以及缓存如何帮忙但又带来新问题”。

#### 必学主干
- String / Hash / List / Set / ZSet
- 缓存旁路模式
- 缓存击穿 / 穿透 / 雪崩
- 分布式锁
- 过期策略
- 持久化基本概念

#### 这一层真正要抓住的 4 个问题
- 为什么不能所有读都打 MySQL？
- 为什么缓存会带来一致性问题？
- 为什么热点 key 会出问题？
- 分布式锁到底锁住了什么？

#### 最小实践
- 给已有项目加入查询缓存
- 实现一个简单的验证码/限流/分布式锁场景
- 模拟缓存击穿并给出解决策略

#### 过关标准
- 你能解释 Redis 在系统中的角色
- 你能说清缓存三大问题及对应处理思路
- 你能把 Redis 正常接入 SpringBoot 项目

#### 暂时别深挖
- Redis 全量底层源码
- 复杂集群部署细节

---

### Phase 6：单体项目接通
**目标**：把 JavaSE + SpringBoot + MySQL + Redis 接成一个完整系统。

#### 推荐进入项目
- Tlias 教学管理系统
- 苍穹外卖

#### 项目里要盯住的不是“功能”，而是这些
- 分层结构
- DTO / VO / Entity 区分
- 权限 / 登录
- 文件上传
- 事务边界
- 缓存使用点
- 公共字段填充
- 统一返回 / 统一异常
- 部署方式

#### 过关标准
- 你能自己画出项目结构图
- 你能说清一条请求如何经过 Controller → Service → Mapper → DB
- 你能修改一个真实模块而不是只会跑通

#### 暂时别深挖
- 一上来追求自己独立从零造完整大项目
- 还没看懂分层就埋头抄代码

---

### Phase 7：Java 高级（JVM / JUC）
**目标**：解释程序运行时行为，而不是只会调用框架。

#### JVM 必学主干
- 类加载
- 运行时数据区
- 对象创建过程
- 垃圾回收基础
- 常见 GC 思想
- OOM / 内存泄漏 / 性能问题定位思路

#### JUC 必学主干
- 线程安全
- synchronized / Lock
- volatile
- CAS
- AQS（知道它是很多并发工具的骨架）
- 线程池
- 并发容器

#### 这一层真正要抓住的 5 个问题
- 对象放哪？线程共享什么？
- 为什么会线程不安全？
- 锁到底在解决什么？
- 线程池为什么比乱开线程强？
- OOM 和卡顿该怎么初步定位？

#### 最小实践
- 写一个线程不安全示例并修复
- 配一个线程池并说明参数意义
- 用工具观察堆内存或线程状态

#### 过关标准
- 你能说清 synchronized、volatile、CAS 分别解决什么
- 你能解释 JVM 为什么重要
- 你能把这层知识和真实线上问题挂上钩

#### 暂时别深挖
- 过度追 GC 调优参数
- 还没建立业务经验就死磕 JVM 面经细枝末节

---

### Phase 8：中间件
**目标**：理解系统之间为什么要异步、解耦、削峰。

#### 必学主干
- RabbitMQ 或 RocketMQ 先选一个打穿
- Kafka 作为高吞吐日志流系统再补
- 消息可靠性
- 重复消费 / 幂等
- 顺序消息
- 延迟消息
- 死信队列

#### 这一层真正要抓住的 4 个问题
- 为什么不用同步调用解决一切？
- MQ 如何帮你削峰填谷？
- 为什么消息会丢、会重、会乱序？
- 消费者如何保证幂等？

#### 最小实践
- 在 SpringBoot 项目里引入一个消息队列场景
- 实现下单后异步发通知/写日志/更新积分等

#### 过关标准
- 你能说清 MQ 的价值不是“增加复杂度”，而是转移耦合和流量压力
- 你能画出生产者、Broker、消费者的基本模型

---

### Phase 9：分布式 / 微服务
**目标**：从单体系统过渡到“多个服务协作”。

#### 必学主干
- 服务拆分原则
- 注册中心 / 配置中心
- 网关
- OpenFeign / RPC
- 限流 / 熔断 / 降级
- 分布式事务基本思路
- 链路追踪、日志、监控
- Docker 化部署

#### 推荐进入项目
- SpringCloud 微服务课程
- 黑马商城 / 微服务实战

#### 这一层真正要抓住的 5 个问题
- 为什么系统拆开会更复杂？
- 配置、调用、故障、事务为什么都变难了？
- 为什么幂等、补偿、最终一致性会出现？
- 网关到底在系统里处于什么位置？
- 微服务什么时候该用，什么时候不该用？

#### 最小实践
- 把一个单体系统拆成 2~3 个服务
- 引入注册中心和网关
- 做一个最小远程调用

#### 过关标准
- 你能说清微服务不是“高级单体”，而是“复杂性位置改变”
- 你能理解服务治理的必要性

## 5. 练习策略

### 练习只做三类
1. **结构识别题**
   - 这个模块在整个后端系统里负责什么？
2. **机制理解题**
   - 为什么需要 IoC / 缓存 / 事务 / MQ？
3. **最小实现题**
   - 用最少代码验证核心机制。

### 不做的低效事
- 大量抄 CRUD 熟悉度题
- 大量背定义
- 没理解分层就盲抄项目
- 还没到项目阶段就做大型业务系统
