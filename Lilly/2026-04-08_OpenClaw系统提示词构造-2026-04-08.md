# OpenClaw 系统提示词构造 vs Claude Code

## OpenClaw 系统提示词结构

OpenClaw 的系统提示词是紧凑的、分段的，包含固定 section：

| Section | 内容 |
|---------|------|
| Tooling | 当前工具列表 + 简短描述 |
| Safety | 简短 guardrail 提醒，避免 power-seeking |
| Skills | 告知模型如何按需加载 skill 指令 |
| OpenClaw Self-Update | 如何运行 config.apply / update.run |
| Workspace | 工作目录 |
| Documentation | OpenClaw docs 本地路径 |
| Workspace Files (injected) | bootstrap 文件内容 |
| Sandbox | 沙箱运行时信息 |
| Current Date & Time | 用户本地时区和时间 |
| Reply Tags | 可选的回复标签语法 |
| Heartbeats | heartbeat 提示和行为 |
| Runtime | host, OS, node, model, thinking level |
| Reasoning | 推理可见性级别 + /reasoning 切换提示 |

## Bootstrap 文件注入机制

OpenClaw 自动注入以下文件到 context window（每个 agent run 都会注入）：

- AGENTS.md
- SOUL.md
- TOOLS.md
- IDENTITY.md
- USER.md
- HEARTBEAT.md

限制：
- 单个文件最大 20,000 chars（bootstrapMaxChars）
- 总 bootstrap 内容最大 150,000 chars（bootstrapTotalMaxChars）
- 超过任一限制会截断并附加警告
- `memory/*.md` 不自动注入，通过 memory_search 按需访问

sub-agent 只注入 AGENTS.md 和 TOOLS.md（最小化 context）。

## promptMode 机制

OpenClaw 为不同类型的 agent 渲染不同大小的系统提示词：

| promptMode | 用途 | 包含内容 |
|------------|------|----------|
| full (默认) | 主 agent | 所有 section |
| minimal | sub-agent | Tooling, Safety, Workspace, Sandbox, Runtime（无 Skills/MemoryRecall/Self-Update 等） |
| none | 极简 | 只有 base identity 行 |

内部通过 `agent:bootstrap` hook 拦截，可以替换 SOUL.md 等文件。

## Skills 注入方式

Skills 不是全量注入，而是注入一个紧凑的列表：

```
<available_skills>
  <skill>
    <name>...</name>
    <description>...</description>
    <location>...</location>
  </skill>
</available_skills>
```

模型通过 `read` 工具按需读取 SKILL.md 文件。这保持 base prompt 小，同时支持按需加载。

## Claude Code 对比

Claude Code 的系统提示词构造更封闭（无公开源码），但已知：

- MEMORY.md 最多 200 行 / 25KB（比 OpenClaw 的 20K char 限制更细）
- 有 auto-compact 机制（token 阈值触发）
- 有多种 compaction 类型：autoCompact/microCompact/snipCompact/sessionMemoryCompact/contextCollapse
- Skills/MCP 工具通过动态发现机制加载，不是静态列表

## 工程意义

**OpenClaw 的设计选择：**
1. 紧凑分段结构 → 易于理解和调试
2. Bootstrap 文件注入 → 保证 agent 始终看到身份和工具定义
3. promptMode → 不同类型 agent 用不同大小的 prompt，节省 token
4. Skills 按需加载 → base prompt 不膨胀
5. 150K total cap → 防止 bootstrap 内容无限增长

**Claude Code 的设计选择：**
1. 更细的 MEMORY.md 限制（200行/25KB）→ 更严格的 context 管理
2. 多种 compaction 类型 → 更灵活的 context 压缩
3. 动态工具发现（MCP）→ 更灵活的扩展机制

**对于中国区 agent 工程：**
- OpenClaw 的 promptMode 设计很实用，可以为不同类型 agent 配置不同 prompt
- Skills 按需加载的思路值得借鉴，避免 base prompt 膨胀
- Bootstrap 文件大小限制和 total cap 是必要的 guardrail，防止意外的高 context 消耗
