# Claude Code 源码架构笔记 - Task 系统\n\n## 源码位置\n\n/mnt/f/agent_project/code_agent/claude_code_src/\n版本：2.1.88（从 npm source map 还原，共 70w 行）\n\n## Task 类型系统\n\nClaude Code 的任务分为 7 种类型：\n\n| TaskType | 含义 |\n|----------|------|\n| local_bash | 本地 bash 命令 |\n| local_agent | 本地 agent 执行 |\n| remote_agent | 远程 agent |\n| in_process_teammate | 进程内 teammate |\n| local_workflow | 本地工作流 |\n| monitor_mcp | MCP 服务器监控 |\n| dream | 某种特殊任务（名字很有意思） |\n\n## Task 状态\n\n- pending\n- running\n- completed\n- failed\n- killed\n\nisTerminalTaskStatus() 判断是否为终态：completed / failed / killed\n\n## TaskStateBase 核心字段\n\n- id, type, status, description\n- toolUseId：关联的 tool use\n- startTime, endTime, totalPausedMs\n- outputFile：任务输出文件路径\n- outputOffset：读取偏移量\n- notified：是否已通知\n\n## TaskHandle\n\n\n\n## TaskContext\n\n\n\nAbortController 用于任务取消，AppState 是全局状态。\n\n## 工程意义\n\nClaude Code 的 task 是有状态对象，不是无状态函数调用。\n这意味着：\n- 任务可以暂停（totalPausedMs）\n- 任务有输出文件持久化（outputFile）\n- 任务可以被 kill（通过 AbortController）\n- 任务状态变化通过 setAppState 驱动 UI 更新\n\n这和 OpenClaw 的 session/agent loop 模型有明显区别。\nClaude Code 的 task 更像"有持久化状态的后台进程"，OpenClaw 更像"事件驱动的函数调用"。\n\n## 来源\n\n/mnt/f/agent_project/code_agent/claude_code_src/src/Task.ts\n
