# Claude Code 安全与权限模型

## Permission 模式

Claude Code 有 4 种 permission mode：

| Mode | 说明 |
|------|------|
| `default` | 交互式确认 |
| `acceptEdits` | 自动允许编辑操作 |
| `plan` | 自动进入计划模式 |
| `bypassPermissions` | 完全绕过权限检查（危险） |

可通过 `permissions.defaultMode` 全局设置，或通过 `initialPermissionMode` 为新对话设置初始模式。

## Permission Rules

支持细粒度规则，语法类似 `Bash(git *)`：

```json
{
  "permissions": {
    "allow": ["Edit(.claude/settings.json)", "Bash(git *)", "Read(/**)"],
    "deny": ["Bash(rm -rf /)"],
    "defaultMode": "default"
  }
}
```

支持目录级别规则（如 `Read(src/**)`）和工具+pattern 组合。

## Hook 与 Permission 交互

从 changelog 发现的关键设计：

1. **PreToolUse Hook** - 工具执行前拦截，可返回 `permissionDecision: "allow"` 配合 `updatedInput`
2. **StopFailure Hook** - API 错误时触发（rate limit, auth failure）
3. **PermissionDenied Hook** - auto mode 分类器拒绝时触发，返回 `{retry: true}` 可重试
4. **Conditional Hooks** - 支持 `if` 字段过滤触发条件，减少不必要的进程创建

## Credential 管理

从 settings.json 看到的配置：

- `ANTHROPIC_AUTH_TOKEN`：认证 token（支持 MiniMax 等第三方提供商）
- `ANTHROPIC_BASE_URL`：API 端点（MiniMax 使用 `https://api.minimaxi.com/anthropic`）
- `CLAUDE_CODE_SUBPROCESS_ENV_SCRUB=1`：剥离 subprocess 环境中的云凭据
- `skipDangerousModePermissionPrompt: true`：跳过危险模式确认提示

## MCP OAuth 支持

- `clientId/callbackPort/authServerMetadataUrl`
- XAA (Cross-App Access) 支持
- token 自动刷新
- 支持 RFC 9728 Protected Resource Metadata discovery

## 安全工程要点

1. **bypassPermissions 危险**：生产环境不应使用，调试专用
2. **Hook 可拦截所有工具调用**：适合审计和合规
3. **Credential 剥离**：SUBPROCESS_ENV_SCRUB 防止子进程泄露云凭据
4. **Permission rule 语法**：支持 glob pattern，灵活度高
5. **MCP OAuth 支持**：适合企业级 MCP server 集成

## 与 OpenClaw 对比

| | Claude Code | OpenClaw |
|---|---|---|
| 权限模型 | allow/deny rules + hook | exec 工具 + approval |
| 第三方 API | 内置支持（Base URL 替换） | 依赖 model 配置 |
| Credential 管理 | 环境变量 + SUBPROCESS_ENV_SCRUB | 无专项文档 |
| MCP 支持 | 完整 OAuth + 动态发现 | skill 静态注册 |
