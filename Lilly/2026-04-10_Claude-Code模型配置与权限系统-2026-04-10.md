## 模型别名体系

| Alias | 行为 |
|-------|------|
| `default` | 清除覆盖，恢复账户推荐模型 |
| `best` | 当前最强者，等效 opus |
| `sonnet` | 最新 Sonnet（当前 4.6）日常编码 |
| `opus` | 最新 Opus（当前 4.6）复杂推理 |
| `haiku` | 快速轻量，简单任务 |
| `sonnet[1m]` | Sonnet + 100 万 token 上下文 |
| `opus[1m]` | Opus + 100 万 token 上下文 |
| `opusplan` | plan 阶段用 opus，执行阶段切换 sonnet |

### opusplan 的设计哲学

planning 需要强推理能力，execution 需要速度和成本效率。分开使用不同模型是聪明的分层策略。

## 权限系统

### 三层权限模型

| 工具类型 | 需要审批 | 不再询问行为 |
|---------|---------|------------|
| 只读（Read/Grep） | 否 | N/A |
| Bash 命令 | 是 | 永久（按项目目录 + 命令） |
| 文件修改（Edit/Write） | 是 | 仅当前 session |

### 权限评估顺序

deny → ask → allow，第一个匹配生效。

### 权限模式

| 模式 | 描述 |
|------|------|
| default | 首次使用时提示 |
| acceptEdits | 自动接受工作目录内的文件编辑 |
| plan | 只读模式 |
| auto | 自动批准 + 后台安全检查 |
| dontAsk | 除非预批准，否则自动拒绝 |
| bypassPermissions | 跳过所有提示（受保护目录除外） |

### 受保护目录

bypassPermissions 仍会提示：`.git`、`.vscode`、`.idea`、`.husky`

管理员可禁用 bypassPermissions。

## 权限规则语法

格式：Tool 或 Tool(specifier)

| 规则 | 效果 |
|------|------|
| Bash | 匹配所有 Bash |
| Bash(npm run build) | 精确匹配 |
| Bash(npm run *) | 通配符 |
| Read(./.env) | 匹配特定文件 |
| WebFetch(domain:example.com) | 匹配特定域名 |

## 工程意义

1. opusplan 按任务阶段分配模型：规划用 opus，执行用 sonnet，成本效率最优
2. availableModels 是企业级模型管控完整方案
3. Bash 审批永久化设计避免重复提示
4. 6 种权限模式覆盖从最严到最松的安全级别
