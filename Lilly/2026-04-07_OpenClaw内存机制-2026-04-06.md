# OpenClaw Builtin Memory 机制笔记

## 关键机制

- MEMORY.md 和 memory/*.md 会被自动分块索引（~400 tokens，80 token overlap）
- 存储在 ~/.openclaw/memory/<agentId>.sqlite
- 文件变更触发 debounced reindex（1.5s 延迟）
- 支持三种检索模式：BM25 keyword、vector embedding、hybrid（混合）
- CJK 语言支持 via trigram tokenization
- 无 embedding provider 时只能做 keyword search

## 对 Lilly 的影响

Lilly 使用 memory_search 时，实际上是在查询这个 SQLite 索引。

写入 MEMORY.md 的内容会被自动分块和索引，适合：
- 改变未来编排决策的知识
- 需要跨 session 召回的模式性内容
- durable orchestration lessons（已验证有效的协调规则）

不适合放在 MEMORY.md 的：
- 大量技术细节（可放在 Obsidian，memory 只保留结论）
- 临时工作备注
- 需要精确格式保留的内容

## 实用命令

- openclaw memory status：检查 memory provider 状态
- openclaw memory index --force：强制重建索引
