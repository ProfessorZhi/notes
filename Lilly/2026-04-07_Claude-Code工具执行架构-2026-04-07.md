# Claude Code 工具执行架构\n\n## 核心发现\n\n### Query Loop 结构\n\nClaude Code 的主循环在 query.ts 里：\n-  无限循环\n- 每次迭代前做 context compaction（snip/microcompact/collapse）\n- 然后调用 API，结果回来后执行工具，再继续循环\n\n### 工具执行（toolOrchestration.ts）\n\n关键并发策略：\n\n1. **Read-only 工具并发执行**：最多 10 个（）\n2. **非只读工具串行执行**：防止状态冲突\n3. **Context modifiers**：并发工具的结果用来修改后续工具的上下文\n\npartitionToolCalls 把工具分成两类 batch：\n- 只读工具：Batch(isConcurrencySafe=true)，可以并行\n- 非只读工具：Batch(isConcurrencySafe=false)，必须串行\n\n### Context 管理\n\nClaude Code 有复杂的上下文管理：\n- snipCompactIfNeeded：裁剪长对话\n- microcompact：按 tool_use_id 做缓存\n- applyCollapsesIfNeeded：折叠历史消息\n\n### 对比 OpenClaw\n\nOpenClaw 工具执行：\n- 工具执行在 agent loop 内\n- 没有看到类似的并发策略文档\n- 更偏向串行执行\n\nClaude Code 的并发策略更适合：\n- 多工具同时调用的场景（如同时搜索多个来源）\n- 需要并行加速 agent 执行\n\n## 工程价值\n\n在设计 agent 系统时：\n- 工具应区分只读/非只读\n- 只读工具可以安全并发\n- 并发数应可配置\n- 非只读工具需要串行以保证状态一致性\n\n## 来源\n\nClaude Code 源码：\n- src/query.ts（主循环）\n- src/services/tools/toolOrchestration.ts（工具执行）\n
