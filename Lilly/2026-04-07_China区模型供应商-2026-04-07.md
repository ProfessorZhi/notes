# China 区模型供应商概览

## OpenClaw 已集成的中国模型供应商

| 供应商 | 模型 ID 格式 | 备注 |
|--------|-------------|------|
| DeepSeek | deepseek/deepseek-chat, deepseek-reasoner | OpenAI-compatible API，V3.2 |
| MiniMax | minimax/MiniMax-M2.7, minimax/M2.7-highspeed | Lilly 当前模型，image-01 图像生成 |
| Moonshot | moonshot/kimi-k2.5, kimi-coding/k2p5 | 需区分两个 provider（API key 不通用） |
| Zhipu/GLM | glm/... | ChatGLM 系列 |
| 百度 Qianfan | qianfan/... | ERNIE 系列 |
| 阿里 Qwen | qwen/... | 通义千问 |
| 火山引擎 | volcengine/... | 字节跳动 |
| 小米 | xiaomi/... | MiLM |

## 各家亮点

- **DeepSeek-reasoner**：推理模型，适合需要 CoT 的任务
- **Moonshot kimi-k2-thinking**：带思考过程的模型，适合复杂分析
- **MiniMax M2.7-highspeed**：高速版本，适合简单任务降成本
- **MiniMax image-01**：图像生成，支持多种比例

## 工程意义

Lilly 作为中国区 agent engineer，需要理解：

1. 各家 API 不兼容，key 不通用，需要分别配置
2. OpenClaw 通过 environment variable 提供 auth，支持 env.shellEnv 或 ~/.openclaw/.env
3. 可以在 agents.defaults.model.fallbacks 里配置多家 provider 作为 failover
4. 中国 provider 通常比海外 provider 延迟更低、性价比更高
5. 推理模型（reasoner/thinking）适合复杂任务但成本更高

## 认知更新

之前对"多模型 failover"的理解只停留在 auth profile rotation，现在理解到：

- 可以在 config 里配置多个 provider 的 model fallback chain
- 不同任务可以用不同 model（比如简单总结用 M2.7-highspeed，复杂推理用 deepseek-reasoner）
- image generation 也有独立的 model 配置项
