# Heartbeat 机制深度笔记\n\n## 关键发现（之前没注意到的）\n\n### 1. isolatedSession\nisolatedSession: true 时，每次心跳在全新 session 运行，不带历史上下文。token 消耗从 ~100K 降到 ~2-5K。\n\n### 2. lightContext\nlightContext: true 时，只注入 HEARTBEAT.md，不加载其他 bootstrap 文件。\n\n### 3. HEARTBEAT_OK 生效位置\n必须出现在回复的**开头或结尾**，中间位置不算。\n\n### 4. HEARTBEAT.md 空文件跳过\n如果 HEARTBEAT.md 只有空行和标题（无实际内容），心跳直接跳过，不消耗 API。\n\n### 5. activeHours\n心跳可限制在特定时间段执行，之外直接跳过，不触发模型调用。\n\n### 6. Manual wake\n可用 ok 立即触发心跳。\n\n### 7. Per-agent 规则\n如果 agents.list 里任意一个 agent 配置了 heartbeat block，**只有这些 agent** 会跑心跳（不是所有 agent 都跑）。\n\n## 成本优化组合拳\n\n| 优化手段 | 效果 |\n|---------|------|\n| isolatedSession: true | ~100K → ~2-5K tokens |\n| lightContext: true | 只加载 HEARTBEAT.md |\n| target: "none" | 不推送，只内部状态更新 |\n| 更便宜的 model | 如 ollama/llama3.2:1b |\n| HEARTBEAT.md 保持短小 | 减少 prompt 体积 |\n\n## HEARTBEAT_OK 行为\n\n- 位置：必须开头或结尾\n- 逻辑：strip HEARTBEAT_OK 后，若剩余内容 ≤ ackMaxChars（默认300），整条消息丢弃\n- 中间出现：不触发特殊逻辑\n\n## 对 Lilly 的影响\n\nLilly 的 heartbeat 当前配置是默认的（30m/1h）。如果想优化成本：\n- 保持 HEARTBEAT.md 短小精炼\n- 继续用 isolatedSession（如果已配置）\n- 继续用 lightContext\n\n## 来源\n\n/home/dministrator/openclaw-wsl/docs/gateway/heartbeat.md\n
