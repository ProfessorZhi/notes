# 多 Agent 稳定性设计原则

## 核心认知

系统稳定性 > 单点能力最优。

04-05 分工方案重构的根本原因：飞书内部文档和知识库访问不稳定，如果把整个协作链建立在这个不稳定依赖上，整条链都会经常卡死。

新方案的第一性原则：

> 不要依赖不稳定能力来构建协作链。先把多 bot 协作跑通在稳定能力上。

## 三个稳定能力

1. **外部信息收集**（绮雯）— 互联网搜索，不依赖内部系统
2. **资料加工整理**（Sasha）— 结构化，不依赖执行类工具
3. **最终交付**（Sasha）— 写作，不依赖内部系统

## 教训

- 在多 agent 系统里，**协作链的强度等于它最弱的那一环**
- 如果某个 bot 依赖的能力本身不稳定，整个系统都会受影响
- 重构应该改架构，而不是让各个 bot 去硬做不稳定的事
