# OpenClaw 插件架构笔记\n\n## 核心概念：Capability Model\n\n插件通过注册能力来扩展 OpenClaw，主要能力类型：\n\n| 能力类型 | 注册方法 | 示例 |\n|---------|---------|------|\n| Text inference | registerProvider | openai, anthropic |\n| CLI inference backend | registerCliBackend | openai, anthropic |\n| Speech | registerSpeechProvider | elevenlabs, microsoft |\n| Media understanding | registerMediaUnderstandingProvider | openai, google |\n| Image generation | registerImageGenerationProvider | openai, google |\n| Web search | registerWebSearchProvider | google |\n| Channel/messaging | registerChannel | msteams, matrix |\n\n## 插件分类（按形状）\n\n- **plain-capability**：只注册一种能力（如 mistral）\n- **hybrid-capability**：注册多种能力（如 openai 同时注册 text/speech/media/image）\n- **hook-only**：只注册 hooks，无 capability\n- **non-capability**：注册 tools/commands/services，但无 capability\n\n## Legacy Hooks\n\n- ：仍支持，但已废弃\n- 推荐迁移到：（模型/ provider override）和 （prompt 变更）\n\n## 兼容性信号（openclaw doctor / plugins inspect）\n\n| 信号 | 含义 |\n|-----|------|\n| config valid | 配置正常 |\n| compatibility advisory | 使用较旧但仍支持的模式 |\n| legacy warning | 使用了已废弃的 before_agent_start |\n| hard error | 配置无效或插件加载失败 |\n\n## 工程价值\n\n- 调试插件：用  看插件形状和 capability\n- Hook 优先级：before_model_resolve > before_prompt_build > before_agent_start\n- 理解当前系统里哪些是 hook-only（legacy），哪些是 capability-based（新）\n\n## 来源\n\n/home/dministrator/openclaw-wsl/docs/plugins/architecture.md\n
