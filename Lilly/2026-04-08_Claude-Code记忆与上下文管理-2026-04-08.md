# Claude Code 记忆与上下文管理\n\n## MEMORY.md 系统\n\n位置：src/memdir/memdir.ts\n\n关键约束：\n- MEMORY.md 最多 200 行（MAX_ENTRYPOINT_LINES）\n- 最多 25KB（MAX_ENTRYPOINT_BYTES）\n- 超过任一限制会截断并附加警告\n- 截断优先按行截断（在换行处中断），再按字节截断\n\n## Auto-Compact 机制\n\n触发条件：\n- effectiveContextWindow - 13,000 tokens（AUTOCOMPACT_BUFFER_TOKENS）\n- 预留 20K tokens 用于摘要输出\n- 可通过 CLAUDE_CODE_AUTO_COMPACT_WINDOW 环境变量覆盖\n\n熔断机制：\n- 连续 3 次 compact 失败后停止重试（MAX_CONSECUTIVE_AUTOCOMPACT_FAILURES = 3）\n- 避免无限 retry 浪费 API 调用\n\n阈值定义：\n- AUTOCOMPACT_BUFFER_TOKENS = 13,000\n- WARNING_THRESHOLD_BUFFER_TOKENS = 20,000\n- ERROR_THRESHOLD_BUFFER_TOKENS = 20,000\n- MANUAL_COMPACT_BUFFER_TOKENS = 3,000\n\n## Compaction 类型\n\n1. **autoCompact**：自动触发，由上下文窗口超限\n2. **microCompact**：按 tool_use_id 做缓存，不检查内容\n3. **snipCompact**：裁剪长对话\n4. **sessionMemoryCompact**：session 级别内存压缩\n5. **contextCollapse**：折叠历史消息\n\n## 与 OpenClaw 的对比\n\n| | Claude Code | OpenClaw |\n|---|---|---|\n| 记忆文件 | MEMORY.md（200行/25KB限制） | MEMORY.md + memory/*.md |\n| 上下文压缩 | 多种机制（auto/micro/snip/collapse） | 依赖 session pruning |\n| 压缩触发 | 自动按 token 阈值 | 需要配置 maintenance |\n\n## 工程意义\n\nClaude Code 的 auto-compact 是主动式管理，不需要等到 context window 满了才压缩。\n这保证了每次 API 调用都有足够的空间，不会因为 context 超限而失败。\n\nOpenClaw 的 session maintenance 是被动式，需要配置规则触发。\n对于长期运行的 agent，主动式压缩更适合。\n\n## 来源\n\nClaude Code 源码：\n- src/memdir/memdir.ts\n- src/services/compact/autoCompact.ts\n- src/services/compact/compact.ts\n
