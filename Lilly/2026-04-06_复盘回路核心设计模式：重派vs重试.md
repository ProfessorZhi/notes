# 复盘回路核心设计模式：重派 vs 重试

## 核心结论
主管 bot 需要把两个判断分开：
1. 结果够不够交付
2. 有没有找对执行 bot

质量问题就返工，同 lane 修缺口；路由问题就改派，换到更合适的专项 bot。

## 为什么重要
如果把返工和改派混成一件事，多 agent 系统会出现两类浪费：
- 对错误的人反复重试
- 对正确的人不给清晰缺口

## 最小工程约束
- 每个任务最多 3 次内部处理，防止无限循环
- 返工必须指出具体缺口，不能只说“重做”
- 改派是独立动作，不等于同 agent 重试
- 主管只做轻量验收，判断“够不够交付”，不追求“世界最佳答案”

## 迁移价值
这条模式适用于多 agent 编排、human-in-the-loop 工作流、以及任何带专项路由的任务系统。
