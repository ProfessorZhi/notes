# Claude Code 工具安全模型\n\n## 文件读取限制\n\n| 限制 | 默认值 |\n|------|--------|\n| maxSizeBytes | 256 KB（总文件大小，非读取部分） |\n| maxTokens | 25,000（实际输出 token 数） |\n\n可通过环境变量 CLAUDE_CODE_FILE_READ_MAX_OUTPUT_TOKENS 覆盖。\n\n## Bash 工具安全模型\n\nClaude Code 使用 **显式 Allowlist + Flag 验证** 的安全模型：\n\n- 维护了一个 CommandConfig 系统\n- 每个命令（fd、ripgrep、git、gh 等）有明确的 safe flags allowlist\n- 比如 fd 的 -x/--exec 和 -X/--exec-batch 被明确排除（因为它们对每个搜索结果执行任意命令）\n\n对比 OpenClaw 的安全模型：\n- OpenClaw 使用 exec 工具，默认允许执行任意命令\n- Claude Code 的模型更严格，但配置成本更高\n\n## readOnlyValidation 设计\n\n\n\n## 工程意义\n\nClaude Code 的安全模型适合：\n- 高安全要求环境\n- 避免 model 误执行危险命令\n\nOpenClaw 的模型适合：\n- 灵活调试场景\n- 降低配置复杂度\n\n如果要在中国区 agent 工程中提高安全性，可以借鉴 Claude Code 的 flag allowlist 思路。\n\n## 来源\n\nClaude Code 源码：\n- src/tools/FileReadTool/limits.ts\n- src/tools/BashTool/readOnlyValidation.ts\n
