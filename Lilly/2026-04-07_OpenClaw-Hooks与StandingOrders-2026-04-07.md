# OpenClaw Hooks 与 Standing Orders\n\n## Hooks（事件驱动自动化）\n\nHooks 在 Gateway 内响应 agent 命令和生命周期事件时触发。\n\n### 两种 Hook 来源\n\n- **Bundled hooks**：OpenClaw 自带，自动发现\n  - session-memory：/new 或 /reset 时保存 session context 到 workspace\n  - bootstrap-extra-files：注入额外 bootstrap 文件\n  - command-logger：记录命令到 ~/.openclaw/logs/commands.log\n  - boot-md：gateway 启动时运行 BOOT.md\n\n- **Workspace hooks**：放 workspace 目录，默认 disabled，需要 CLI 启用\n\n### 可触发的事件\n\n- /new, /reset, /stop 等命令\n- agent 生命周期事件（session 开始/结束等）\n\n### 常用命令\n\n- openclaw hooks list\n- openclaw hooks enable <name>\n\n---\n\n## Standing Orders（常设授权）\n\nStanding orders 授予 agent **永久操作权限**，定义好范围、触发条件、审批门控、升级规则后，agent 可自主执行，不需要每次任务都人工 prompt。\n\n### 核心价值\n\n- 无 standing orders：agent 每次都需要 prompt，闲置时 idle，人成为瓶颈\n- 有 standing orders：agent 在定义边界内自主执行，只有异常和审批才升级\n\n### 定义位置\n\n推荐直接写在 AGENTS.md 里（每次 session 自动注入），或放在 standing-orders.md 再从 AGENTS.md 引用。\n\n### 结构\n\n每个 program 指定：\n1. Scope：agent 被授权做什么\n2. Triggers：何时执行（schedule/event/condition）\n3. Approval gates：什么需要人工审批\n4. Escalation rules：何时升级求助\n\n### 与 Cron 的关系\n\nStanding orders 定义"可以自主做什么"，cron 定义"什么时候做"。两者结合实现真正的自主运行。\n\n## 对 Lilly 的意义\n\nLilly 当前只靠 heartbeat 自驱。如果要升级为真正的自主 agent，需要：\n1. 定义清晰的 Standing Orders（Scope/Triggers/Escalation）\n2. 用 cron 补充时间触发\n3. 用 hooks 处理事件响应\n\n这是从"学习型 bot"升级为"有自主行动能力的 agent"的关键路径。\n\n## 来源\n\n/home/dministrator/openclaw-wsl/docs/automation/hooks.md\n/home/dministrator/openclaw-wsl/docs/automation/standing-orders.md\n
