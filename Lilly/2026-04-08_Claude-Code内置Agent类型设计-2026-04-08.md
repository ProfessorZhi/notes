# Claude Code 内置 Agent 类型设计

## AgentDefinition 结构

```typescript
interface BuiltInAgentDefinition {
  agentType: string
  whenToUse: string           // 何时使用（AgentTool 选项提示）
  tools: string[] | ['*']   // 工具列表，* = 全部
  disallowedTools?: string[] // 禁用工具（deny-list）
  source: 'built-in' | 'user' | 'project'
  baseDir: string
  model?: string              // 空/ inherit = 继承父 agent
  getSystemPrompt?(): string  // 动态生成系统提示
  omitClaudeMd?: boolean      // 从上下文省略 CLAUDE.md
  criticalSystemReminder_EXPERIMENTAL?: string  // 额外关键提醒
  background?: boolean        // 后台运行
  color?: string              // UI 颜色标识
}
```

## 5 种内置 Agent

| Agent | 模型 | 工具策略 | 关键特点 |
|-------|------|---------|---------|
| **Explore** | haiku (外部) / inherit (ant) | 只读 + deny-list | 快速搜索，parallel 工具调用，强调速度 |
| **Plan** | inherit | 只读 + deny-list | 读 Only 架构规划，输出关键文件列表 |
| **General Purpose** | inherit (默认) | `['*']` 全开 | 通用任务，简洁报告 |
| **Verification** | inherit | 只读 + deny-list | 对抗性验证，必须有命令输出，输出 VERDICT: PASS/FAIL/PARTIAL |
| **claudeCodeGuideAgent** | - | - | 帮助引导 |

## 关键设计模式

### deny-list 而非 allow-list

`disallowedTools` 是 Claude Code 的主要权限限制方式：
```typescript
disallowedTools: [
  AGENT_TOOL_NAME,           // 不能 spawn 子 agent
  EXIT_PLAN_MODE_TOOL_NAME,
  FILE_EDIT_TOOL_NAME,       // 不能编辑
  FILE_WRITE_TOOL_NAME,      // 不能写文件
  NOTEBOOK_EDIT_TOOL_NAME,
]
```

**为什么用 deny-list 而非 allow-list？**
- allow-list 限制死所有工具，工作量巨大（每个工具都要明确放行）
- deny-list 只管住关键危险操作，其余工具默认可用
- 对于"只读探索"类任务，4 个编辑工具deny 即可，搜索/读文件工具全部放行

### Verification Agent 的对抗性设计

Verification agent 是最有特色的：
1. **明确的失败模式描述**：验证回避（只读代码不运行）、被前 80% 迷惑
2. **必须运行命令**：每个 check 必须有 `**Command run:**` 和 `**Output observed:**`，读代码不算 PASS
3. **对抗性探测要求**：至少包含一个边界测试（并发、边界值、幂等性、孤儿操作）
4. **VERDICT 输出格式**：严格格式 `VERDICT: PASS/FAIL/PARTIAL`， caller 解析这个字符串

### model 继承策略

- `model: 'inherit'` = 使用父 agent 的模型
- `model: 'haiku'` = 使用最便宜的模型（Explore 用）
- 外部用户用 haiku 保证速度，ants 用 inherit 保持能力

### omitClaudeMd 节省 token

Plan 和 Explore 都设置 `omitClaudeMd: true`，因为只读任务不需要项目规范。

## 工程意义

1. **专用 agent 比通用 agent 更可靠**：Explore/Plan/Verification 各有固定边界，不会做超出范围的事
2. **deny-list 是最小权限原则的好选择**：只禁用真正危险的工具，其余全部可用，比枚举所有可用工具更省力
3. **Verification agent 的核心价值**：强制要求命令执行证据，而不是"代码看起来对"——这对 China 区 LLM 应用工程的代码验收有直接参考价值
4. **background flag**：后台 agent 不阻塞主会话，支持并行验证
