# Claude Code 多 Agent 模式\n\n## 两种多 Agent 架构\n\n### 1. Coordinator Mode\n\n专门的 orchestrator agent，负责：\n- 理解任务\n- 拆解分发\n- 验收结果\n- 返工/改派\n\n其他 agent 是被协调的工作节点。\n\n### 2. Fork Subagent\n\n隐式分支：\n- 子 agent 继承父 agent 的完整对话上下文和系统 prompt\n- 所有 agent spawn 运行在后台（async），通过 unified task notification 交互\n- /fork <directive> 命令可触发\n\n**限制**：Recursive forking 被禁止（检测 fork boilerplate tag）\n\n**特点**：\n- subagent_type 省略时触发\n- 工具池完全继承（tools: ['*']）\n- model 默认 inherit（继承父的 model）\n- permissionMode: 'bubble'（权限提示冒泡到父 terminal）\n\n## 与 OpenClaw 的关键区别\n\n| | Claude Code | OpenClaw |\n|---|---|---|\n| 多 agent 模式 | Coordinator + Fork | 多 workspace + routing |\n| 子 agent 上下文 | Fork 继承全量上下文 | 每个 agent 独立 session |\n| 通信模型 | 共享 context + task notification | 消息路由 |\n| 权限模型 | bubble（权限提示父） | 工具策略 |\n\nClaude Code 的 Fork 模型更像"进程分支"，上下文共享程度极高。\nOpenClaw 的每个 agent 是完全独立的 session。\n\n## 工程意义\n\n如果要设计多 agent 协作：\n- Claude Code 适合"高度协作、上下文强共享"场景\n- OpenClaw 适合"独立 agent、消息路由"场景\n- 中国的多 bot 方案（nami 主管）更接近 OpenClaw 的路由模型\n\n## 来源\n\nClaude Code 源码：\n- src/coordinator/coordinatorMode.ts\n- src/tools/AgentTool/forkSubagent.ts\n- src/tools/AgentTool/runAgent.ts\n
