# Claude Code 记忆与上下文系统

## 信息来源

- https://code.claude.com/docs/en/memory
- https://code.claude.com/docs/en/context-window（可视化模拟）

## 两种记忆系统

| | CLAUDE.md | Auto Memory |
|--|-----------|-------------|
| 谁编写 | 人 | Claude 自己 |
| 内容 | 指令和规则 | 从纠正中学到的模式和偏好 |
| 范围 | 项目/用户/组织 | 每个工作树 |
| 加载量 | 全部（随上下文加载） | 前 200 行或 25KB（取较小值） |
| 用途 | 编码标准、工作流、架构 | 构建命令、调试洞察、偏好 |

**两者都是上下文，不是强制配置**——写得更具体、更简洁，Claude 遵循得更一致。

## CLAUDE.md 分层体系

| 范围 | 位置 | 用途 |
|------|------|------|
| Managed policy | `/etc/claude-code/CLAUDE.md`（Linux/WSL） | 组织级，IT/DevOps 管理 |
| Project | `./CLAUDE.md` 或 `./.claude/CLAUDE.md` | 团队共享，项目级标准 |
| User | `~/.claude/CLAUDE.md` | 个人偏好，所有项目 |
| Local | `./CLAUDE.local.md` | 个人项目特定，排除 git |

更具体的位置覆盖更宽泛的位置。

## 路径规则：`.claude/rules/`

大型项目中可以将指令拆分到 topic-specific 文件，用路径模式匹配：
- `api-conventions.md`：匹配 `src/api/**`
- `testing.md`：匹配 `*.test.ts`

路径规则在 Claude 读取匹配的文件时**自动加载**，不在启动时全部加载。

## 上下文窗口启动填充（实测数据）

来自 context-window 可视化的 token 预算：

| 内容 | Token 数 | 说明 |
|------|---------|------|
| System prompt | 4,200 | 核心行为指令，始终加载 |
| Auto memory (MEMORY.md) | 680 | 前 200 行或 25KB |
| Environment info | 280 | 目录、平台、shell、OS 版本、git 信息 |
| MCP tools（延迟） | 120 | 只有名称，schema 按需加载 |
| Skill descriptions | 450 | 只有描述，完整内容按需加载 |
| ~/.claude/CLAUDE.md | 320 | 全局偏好 |
| Project CLAUDE.md | 1,800 | 项目约定 |

**文件读取主导上下文使用**。示例：一个 auth.ts 读取 = 2,400 tokens。

## 关键设计：按需加载

**Skill 描述只在启动时加载一行描述**
完整 skill 内容**只在实际使用时**加载。设置了 `disable-model-invocation: true` 的 skill 完全不加载，直到用户用 `/name` 调用。

**MCP tool schema 延迟加载**
默认只列出工具名称，schema 在需要时通过 tool search 加载。可通过 `ENABLE_TOOL_SEARCH=auto` 改为在上下文 10% 以内时全部加载。

**路径规则按需触发**
匹配当前读取文件路径的规则才加载，不触发全部加载。

## 三层压缩中的记忆行为

- Skill descriptions 列表**不会在 `/compact` 后重新注入**，只有实际调用的 skill 会被保留
- CLAUDE.md 和路径规则在压缩后重新注入

## 工程意义

1. **200 行是 CLAUDE.md 的软上限**：超过后遵循度下降。长指令应该迁移到 skill 或路径规则。
2. **研究密集型任务用 subagent**：避免大文件读取撑爆主上下文（一个 auth.ts = 2,400 tokens）
3. **路径规则比全局规则更高效**：只匹配时加载，不污染全局上下文
4. **MCP tool schema 延迟加载是标配**：工具数量多时避免启动时超载
5. **hook 输出用 `additionalContext` 传回 Claude**：纯 stdout 不进入上下文（写入 debug log）

## 与 OpenClaw 对比

| 维度 | Claude Code | OpenClaw |
|------|------------|---------|
| 项目级记忆 | CLAUDE.md（分层） | workspace AGENTS.md/SOUL.md |
| 用户级记忆 | ~/.claude/CLAUDE.md | user-level 无明确对等 |
| 路径规则 | .claude/rules/ | 无明确对等 |
| 启动加载 | 延迟加载（skill/MCP） | skills 启动时快照 |
| 压缩策略 | skill descriptions 特殊处理 | compaction 通用处理 |
