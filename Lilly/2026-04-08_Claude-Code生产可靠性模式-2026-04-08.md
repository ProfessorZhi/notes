# Claude Code 生产可靠性模式 - Changelog  Architectural Patterns

## 1. Autocompact Thrash Loop 检测

Claude Code 2.1.88+ 检测 autocompact thrash loop：
- 当连续 3 次 compaction 后上下文立即又填满到限制
- 此时停止重试，报可操作的错误而不是继续浪费 API 调用
- 这是一个"熔断机制"

工程意义：对于长时间运行的 agent，应该有类似的检测机制，防止无限循环浪费资源。

## 2. Hook 输出处理

Claude Code 2.1.88+：
- Hook 输出超过 50K 字符时，保存到磁盘而不是直接注入 context
- 提供文件路径 + 预览，而不是完整内容

工程意义：防止 hook 输出污染 context，这个阈值（50K）值得参考。

## 3. Prompt Cache 稳定性

Claude Code 2.1.88+：
- 发现 tool schema bytes 在 session 期间变化会导致 prompt cache miss
- 修复了这个问题

工程意义：如果要优化 prompt cache，需要注意 tool schema 的稳定性。

## 4. Nested CLAUDE.md 重复注入 Bug

Claude Code 2.1.88+：
- 修复了 nested CLAUDE.md 在长 session 中被重复注入数十次的问题

工程意义：在设计 context 注入系统时，要注意防止"累积性注入膨胀"。

## 5. Conditional Hooks

Claude Code 2.1.85+：
- Hook 支持 `if` 字段，使用 permission rule 语法（如 `Bash(git *)`）
- 可以过滤 hook 何时运行，减少进程创建开销

工程意义：精细化的 hook 触发条件比全局 hook 更高效。

## 6. TaskCreated Hook

Claude Code 2.1.88+：
- 新增 TaskCreated hook 事件
- 有阻塞行为

工程意义：task 创建时可以有精确的拦截点。

## 7. Extended Thinking 默认关闭

Claude Code 2.1.88+：
- thinking summaries 默认不再在交互 session 中生成
- 需要 `showThinkingSummaries: true` 显式开启

工程意义：thinking summary 会产生额外 token 开销，默认关闭是合理的成本控制。

## 8. 多 Element Error Content Bug

Claude Code 2.1.85+：
- 修复了 MCP tool errors 只返回第一个 content block 的问题
- 现在正确返回所有 error content elements

工程意义：MCP error 处理要支持多元素返回。

## 9. Permission Bubble 模型

从早期版本就知道的 fork subagent 特性：
- subagent 的权限提示冒泡到父 terminal
- 统一的权限决策点

## 10. Memory Filename 可点击

Claude Code 2.1.86+：
- "Saved N memories" 通知中的 memory 文件名可点击

工程意义：通知中的文件路径应该是可操作的，而不是纯文本。
