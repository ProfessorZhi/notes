# OpenClaw 上下文管理\n\n## Context 组成\n\n1. System prompt（OpenClaw 构建）：rules、tools、skills、time/runtime、workspace files\n2. Conversation history：messages + assistant messages\n3. Tool calls/results + attachments\n\n## Bootstrap 文件限制\n\n- 每个 workspace 文件最大 ~20,000 chars（超过会被截断）\n- TOOLS.md 被截断过：raw 54,210 → injected 20,962\n- HEARTBEAT.md 如果为空会被标记 MISSING\n\n## Compaction vs Pruning\n\n| | Compaction | Pruning |\n|---|---|---|\n| 做什么 | 把旧对话总结成摘要 | 只裁剪旧工具结果 |\n| 保存吗 | 是（存在 session transcript） | 否（只在内存里） |\n| 范围 | 整个对话 | 只针对工具结果 |\n\n## Auto-compaction\n\n- 默认开启\n- 当 session 接近 context 限制时触发\n- 或者 model 返回 context-overflow error 时触发（compacts 然后 retry）\n- compaction 前会提醒 agent 保存重要笔记到 memory\n\n## /context list 输出示例\n\n显示每个 workspace 文件的：\n- raw size vs injected size\n- 是否被截断（TRUNCATED）\n- session tokens：14,250 total / ctx=32,000\n\n## 工程意义\n\n- TOOLS.md 很长但只注入 20K chars，超出部分被丢弃\n- HEARTBEAT.md 如果为空，injected 为 0\n- 保持 workspace 文件短小很重要（超过限制会被截断）\n\n## 来源\n\n/home/dministrator/openclaw-wsl/docs/concepts/context.md\n/home/dministrator/openclaw-wsl/docs/concepts/compaction.md\n
