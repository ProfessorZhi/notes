# Claude Code MCP 集成架构\n\n## MCP 工具包装\n\nMCPTool 是一个通用模板工具，实际的 tool name、description、call 实现都在 mcpClient.ts 里动态覆盖。\n\n关键设计：\n- inputSchema 用 z.object({}).passthrough() 接受任意 JSON\n- maxResultSizeChars = 100,000（比 FileReadTool 的 256KB 大很多）\n- checkPermissions() 可被 MCP server 自定义\n\n## MCP Client 传输类型\n\n| Transport | 说明 |\n|-----------|------|\n| stdio | 本地子进程 |\n| SSE | Server-Sent Events |\n| SSE-IDE | IDE 扩展专用 |\n| HTTP | Streamable HTTP |\n| WS | WebSocket |\n| SDK | SDK 控制 |\n\n## MCP OAuth 支持\n\n- clientId/callbackPort/authServerMetadataUrl\n- XAA (Cross-App Access) 支持\n- token 自动刷新\n\n## 工具发现\n\nMCP 服务器在连接时通过 ListToolsResult 提供工具列表，Claude Code 动态加载为 MCPTool 实例。\n\n## 工程意义\n\nClaude Code 的 MCP 集成是"瘦代理"模型：\n- MCP 服务器定义工具 schema 和处理逻辑\n- Claude Code 只负责传输和展示\n- 工具发现是动态的，不是编译时绑定的\n\n这和 OpenClaw 的 skill 系统不同：\n- OpenClaw skill 是静态注册\n- Claude Code MCP 是动态发现\n\n动态发现更适合工具数量多、经常变化的场景。\n\n## 来源\n\nClaude Code 源码：\n- src/services/mcp/client.ts\n- src/services/mcp/types.ts\n- src/tools/MCPTool/MCPTool.ts\n
