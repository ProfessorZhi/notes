# Claude Code 任务系统架构\n\n## 7 种任务类型\n\n| TaskType | 前缀 | 说明 |\n|----------|------|------|\n| LocalShellTask | b | 本地 bash 命令 |\n| LocalAgentTask | a | 本地 agent |\n| RemoteAgentTask | - | 远程 agent |\n| InProcessTeammateTask | - | 进程内 teammate |\n| LocalWorkflowTask | - | 本地工作流 |\n| MonitorMcpTask | - | MCP 服务器监控 |\n| DreamTask | - | dream 任务 |\n| LocalMainSessionTask | s | 主会话后台任务（Ctrl+B x2） |\n\n## Task ID 生成\n\n- LocalShellTask：'b' 前缀\n- LocalAgentTask：'a' 前缀\n- LocalMainSessionTask：'s' 前缀（8位字母数字）\n\n## Stall Watchdog\n\n当 bash 命令 45 秒无输出时触发检查：\n- 如果最后输出像交互式提示符（y/n, Press any key 等），发送通知\n- 防止误报"已完成"\n- 对于 git log -S、长时间构建等会跳过（因为不是交互式提示）\n\n## 任务状态\n\n- pending / running / completed / failed / killed\n- isTerminalTaskStatus()：completed / failed / killed 是终态\n\n## 后台任务通知\n\n- 任务完成时发送 TASK_NOTIFICATION_TAG\n- SDK consumers 不会因为缺少 STATUS 标签而误判任务状态\n\n## 工程意义\n\nClaude Code 的任务系统是完整的后台任务管理：\n- 任务 ID 追踪\n- 输出文件持久化\n- stall 检测\n- 取消/终止机制\n- 与主会话的隔离\n\n每个任务有独立的输出文件和状态机，这是比简单函数调用更重但更可控的模型。\n\n## 来源\n\nClaude Code 源码：\n- src/Task.ts\n- src/tasks/types.ts\n- src/tasks/LocalShellTask/LocalShellTask.tsx\n- src/tasks/LocalMainSessionTask.ts\n
