# 飞书MCP接入完整教程

## 什么是飞书MCP

MCP（Model Context Protocol，模型上下文协议）是一种让AI助手能够调用外部工具的标准化协议。飞书MCP接入后，可以让Claude Code等AI编程工具直接操控飞书文档、多维表格、群组消息等飞书功能，实现真正的"AI+飞书"协同办公。

## 整体流程概览

```
创建飞书应用 → 添加机器人能力 → 配置权限 → 发布应用 → 安全设置
→ 安装依赖 → 用户身份登录 → Claude Code集成 → 测试使用
```

---

## 第一步：在飞书开发者后台创建应用

### 1.1 访问开发者后台

打开 **飞书开放平台**：https://open.feishu.cn

### 1.2 创建企业自建应用

1. 登录后点击「创建企业自建应用」
2. 输入应用名称（例如：`飞书MCP测试`）
3. 点击「创建」

### 1.3 获取凭证信息

创建完成后，在应用的「凭证与基础信息」页面获取：

| 凭证类型 | 用途 |
|---------|------|
| `App ID` | 应用的唯一标识，后续配置MCP需要用到 |
| `App Secret` | 应用密钥，用于身份验证 |

---

## 第二步：添加机器人能力

1. 在应用详情页，点击左侧「添加应用能力」
2. 找到「机器人」选项，点击添加

---

## 第三步：开通权限

飞书MCP需要多种权限才能正常工作。需要在两个地方都开通：**应用权限** 和 **用户权限**。

### 3.1 开通应用权限

进入「权限管理」→「应用权限」，开通以下权限：

| 权限类别 | 需要的权限 |
|---------|-----------|
| 云文档 | 选中所有云文档相关权限 |
| 消息与群组 | 选中所有消息相关权限 |
| 多维表格 | 选中所有多维表格权限 |

### 3.2 开通用户权限

进入「权限管理」→「用户权限」，同样开通：

| 权限类别 | 需要的权限 |
|---------|-----------|
| 云文档 | 选中所有云文档相关权限 |
| 消息与群组 | 选中所有消息相关权限 |
| 多维表格 | 选中所有多维表格权限 |
| 通讯录 | 选中相关权限（发送消息需要获取用户ID）|

> ⚠️ **注意**：应用权限和用户权限都要开通，不能只开一个。

---

## 第四步：发布应用

1. 点击「创建版本」
2. 填写版本号（如 `1.0.0`）
3. 点击「保存」
4. 点击「申请线上发布」
5. 需要企业管理员审批通过（如果是企业自建应用）

审批通过后，页面会显示「已发布」状态，此时应用可以正常使用。

---

## 第五步：安全设置（添加重定向URL）

### 5.1 为什么需要添加

在后续步骤执行用户身份登录时，如果未配置重定向URL，会报错 `redirect_uri illegal`（重定向URL不合法）。

### 5.2 配置步骤

1. 在应用页面找到「安全设置」
2. 找到「重定向URL」配置项
3. 添加以下URL：

```
http://localhost:3000/callback
```

> ⚠️ 注意：`/callback` 路径不能省略，这是MCP登录验证必需的回调地址。

4. 点击「添加」→ 「确认」

> ✅ **优点**：添加重定向URL后**不需要重新发布应用**，即时生效。

---

## 第六步：安装依赖

### 6.1 确认Node.js已安装

```bash
node --version
```

如果未安装，需要先安装 Node.js。

### 6.2 安装open-mcp

```bash
npm install -g open-mcp
```

---

## 第七步：用户身份登录

这一步是将你的飞书账号与刚创建的应用绑定，让MCP知道"是谁在使用"。

### 7.1 执行登录命令

```bash
npx openmcp authenticate \
  --app-id=你的AppID \
  --app-secret=你的AppSecret
```

**示例**：

```bash
npx openmcp authenticate \
  --app-id=cli_xxxxxxxxxxxxx \
  --app-secret=xxxxxxxxxxxxxxxxxxxxxxxx
```

### 7.2 授权流程

执行命令后会弹出浏览器窗口，提示你登录飞书账号并授权给该应用。授权成功后，终端会显示：

```
Success!
```

---

## 第八步：在Claude Code中集成飞书MCP

### 8.1 添加MCP服务器配置

执行以下命令（注意添加 `--global` 参数可以让配置作用于全局，所有项目都能使用）：

```bash
claude mcp add jsonrpc-飞书MCP \
  --name "feishu-mcp" \
  --command "npx" \
  --args "-y openmcp run feishu-mcp --app-id=你的AppID --app-secret=你的AppSecret"
```

> ⚠️ 如果配置时出现语法错误，注意在 JSON 配置的前后添加单引号。

### 8.2 验证配置

```bash
claude mcp list
```

应该看到类似输出：

```
feishu-mcp - connected
```

---

## 第九步：测试使用

### 9.1 创建飞书文档

在Claude Code中输入：

```
帮我创建一个飞书文档，标题是"如何追回前任的心"，然后帮我总结一下文档内容，列出行动清单
```

> 💡 首次使用可能需要确认权限，加上 `--dangerously-skip-permission-prompts` 参数可跳过确认。

### 9.2 爬取数据整理到多维表格

```
帮我爬取秋知2046最新的20条笔记数据，整理到飞书多维表格
```

### 9.3 发送群消息

```
帮我在相亲相爱一家人群里面发一条消息，内容是"家人们早上好"
```

> 💡 发送消息时如果提示权限不足，终端会显示权限申请链接，点击即可直接跳转到权限开通页面。

## 常见问题与注意事项

### 权限相关

- **发消息需要用户权限**：如果发送消息失败，需要在「用户权限」中开通「获取用户ID」相关权限
- **权限即时生效**：在权限管理页面开通权限后**不需要重新发布**，点击终端中的链接可直接跳转开通

### URL配置

- **重定向URL必须包含 `/callback`**：`http://localhost:3000/callback` 是完整的配置格式
- **添加后即时生效**：不需要重新发布应用

### 配置级别

- 不加 `--global` 参数：只对当前项目生效
- 加 `--global` 参数：对当前电脑所有项目生效，推荐使用全局配置
