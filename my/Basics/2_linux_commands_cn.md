# Linux 常用指令速查与详解

覆盖文件目录、文本处理、权限、进程系统、网络与打包这几类高频命令。

> 使用建议：Linux 命令真正重要的不是“背多少”，而是先确认当前目录、目标文件、权限边界，再执行会改动系统状态的命令。涉及删除、覆盖、提权时尤其如此。

## 第二部分：逐条详解

### 文件与目录

#### `pwd`
- **功能**：打印当前 shell 工作目录，先确认自己站在哪，再做复制、删除、打包等操作。
- **常见场景**：进入服务器后先定位目录，避免命令打到错误位置。
- **示例**：
```bash
pwd
```

#### `ls`
- **功能**：查看目录里的文件和子目录，常和 -l、-a、-h 等选项一起用。
- **常见场景**：先看看这个目录里有什么，再决定进入、复制还是删除。
- **示例**：
```bash
ls -lah
```

#### `cd`
- **功能**：改变 shell 的当前位置，后续大部分相对路径命令都会以这里为基准。
- **常见场景**：进入项目目录、日志目录、配置目录时都会用到。
- **示例**：
```bash
cd /var/log
```

#### `mkdir`
- **功能**：创建一个或多个目录；配合 -p 可连父目录一起创建。
- **常见场景**：初始化项目结构、准备备份目录、创建临时输出目录。
- **示例**：
```bash
mkdir -p backup/2026-04-09
```

#### `touch`
- **功能**：文件不存在就创建，存在则更新修改时间。
- **常见场景**：快速占位一个配置文件、脚本文件、日志文件。
- **示例**：
```bash
touch .env.example
```

#### `cp`
- **功能**：把源文件复制到目标位置；复制目录时常加 -r。
- **常见场景**：备份配置文件、拷贝模板、复制静态资源。
- **示例**：
```bash
cp -r src/ src.bak/
```
- **注意**：覆盖风险高，批量复制前先确认目标路径。

#### `mv`
- **功能**：把文件挪到新位置，或者在同目录下直接改名。
- **常见场景**：整理目录、给文件改名、移动构建产物。
- **示例**：
```bash
mv app.conf app.conf.bak
```

#### `rm`
- **功能**：删除对象；删目录常配合 -r，强制删除常配合 -f。
- **常见场景**：清理临时文件、删除旧构建目录、清空错误生成物。
- **示例**：
```bash
rm -rf dist/
```
- **注意**：这是最需要谨慎的命令之一，尤其是 root 权限下的 rm -rf。

#### `cat`
- **功能**：适合快速看短文件，也常用于把多个文件串起来输出。
- **常见场景**：看配置文件、查看简单脚本、与管道组合处理文本。
- **示例**：
```bash
cat /etc/hosts
```

#### `less`
- **功能**：比 cat 更适合看大文件，支持上下翻页、搜索、退出。
- **常见场景**：阅读长日志、配置文件、代码生成结果。
- **示例**：
```bash
less /var/log/syslog
```

#### `tail -f`
- **功能**：持续观察日志最后几行，并在有新内容时自动刷新。
- **常见场景**：盯服务日志、看容器输出、排查线上报错。
- **示例**：
```bash
tail -f /var/log/nginx/error.log
```

### 查找与文本处理

#### `find`
- **功能**：在目录树里递归搜索文件，是定位“文件在哪”的主力命令。
- **常见场景**：找某个配置文件、找最近修改过的日志、批量处理特定后缀文件。
- **示例**：
```bash
find . -name "*.log"
```

#### `grep`
- **功能**：从文件或标准输出中找出匹配内容；配合 -r 可递归查目录。
- **常见场景**：搜错误码、搜函数名、搜配置项。
- **示例**：
```bash
grep -R "DATABASE_URL" .
```

#### `awk`
- **功能**：适合处理“空格或分隔符分列”的结构化文本，能筛行、取列、汇总。
- **常见场景**：统计日志字段、提取命令输出中的某一列。
- **示例**：
```bash
ps aux | awk '{print $2, $11}'
```

#### `sed`
- **功能**：常用于替换字符串、删除行、批量改配置。
- **常见场景**：批量把旧域名改成新域名、调整配置文件中的键值。
- **示例**：
```bash
sed -i 's/8080/9090/g' app.conf
```
- **注意**：直接改文件时先留备份更稳。

#### `sort`
- **功能**：把输入按字典序、数值或指定字段排序。
- **常见场景**：整理统计结果、给日志片段排序、为 uniq 做预处理。
- **示例**：
```bash
sort users.txt
```

#### `uniq`
- **功能**：只处理相邻重复项，所以经常和 sort 连用。
- **常见场景**：统计重复 IP、重复错误码、重复用户名。
- **示例**：
```bash
sort access.log | uniq -c
```

#### `wc`
- **功能**：快速知道一个文件或命令输出的规模。
- **常见场景**：数日志行数、数代码行、确认筛选结果数量。
- **示例**：
```bash
wc -l access.log
```

### 权限与用户

#### `chmod`
- **功能**：调整读、写、执行权限，决定谁能访问或执行一个文件。
- **常见场景**：给脚本加执行权限、收紧密钥文件权限。
- **示例**：
```bash
chmod 755 deploy.sh
```
- **注意**：权限开太大是安全问题，尤其是密钥和配置文件。

#### `chown`
- **功能**：把文件所有权交给正确用户或用户组。
- **常见场景**：部署后修正上传文件归属、让服务用户拥有运行目录。
- **示例**：
```bash
chown -R www-data:www-data /var/www/app
```

#### `sudo`
- **功能**：让普通用户临时以管理员权限运行命令。
- **常见场景**：安装软件、改系统配置、管理服务。
- **示例**：
```bash
sudo systemctl restart nginx
```
- **注意**：sudo 不是万能开关；你要先知道命令本身会改什么。

### 进程与系统

#### `ps`
- **功能**：列出进程状态、PID、命令名等，常与 grep 组合查目标进程。
- **常见场景**：确认服务是否在跑、定位僵尸进程、找到要 kill 的 PID。
- **示例**：
```bash
ps aux | grep nginx
```

#### `kill`
- **功能**：最常见是终止指定 PID，也能发送重载、挂起等其他信号。
- **常见场景**：结束卡死进程、优雅重启前发信号、脚本控制进程生命周期。
- **示例**：
```bash
kill -9 12345
```
- **注意**：-9 很猛，优先考虑正常终止信号，再考虑强杀。

#### `df -h`
- **功能**：按文件系统维度看总量、已用、剩余空间。
- **常见场景**：排查磁盘爆满、确认挂载盘容量。
- **示例**：
```bash
df -h
```

#### `du -sh`
- **功能**：按目录维度看体积，适合找出谁把磁盘吃满了。
- **常见场景**：定位哪个目录最占空间、清理缓存前做判断。
- **示例**：
```bash
du -sh /var/log/*
```

#### `free -h`
- **功能**：快速看机器内存是否吃紧。
- **常见场景**：服务变慢、怀疑 OOM、评估机器负载时。
- **示例**：
```bash
free -h
```

#### `uname -a`
- **功能**：输出系统内核、架构、主机等信息，适合快速识别环境。
- **常见场景**：确认是哪个 Linux 发行版内核、判断兼容性问题。
- **示例**：
```bash
uname -a
```

#### `systemctl`
- **功能**：用于启动、停止、重启、开机自启和查看服务状态。
- **常见场景**：管理 nginx、docker、redis、应用服务。
- **示例**：
```bash
systemctl status nginx
```

#### `journalctl -u`
- **功能**：按服务单元读取日志，比全局系统日志更聚焦。
- **常见场景**：服务启动失败、重启循环、健康检查异常时定位原因。
- **示例**：
```bash
journalctl -u nginx -n 100 --no-pager
```

### 网络与打包

#### `curl`
- **功能**：可以抓页面、调 API、带请求头、发 JSON、下载内容。
- **常见场景**：测试接口连通性、模拟请求、检查健康检查接口。
- **示例**：
```bash
curl -X GET https://example.com/health
```

#### `tar`
- **功能**：Linux 下最常见的打包工具，常配合 gzip 形成 .tar.gz。
- **常见场景**：备份目录、发布构建包、解压交付文件。
- **示例**：
```bash
tar -czf backup.tar.gz ./project
```
- **注意**：-c 是打包，-x 是解包，参数别反。
