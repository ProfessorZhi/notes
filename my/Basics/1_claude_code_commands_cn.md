# Claude Code 常用指令速查与详解

先速查，后详解。适合终端日常使用、会话管理、项目初始化、MCP 与插件扩展。

> 整理说明：内建命令与可见性会随版本、平台、套餐而变化；本文件优先保留最常用、最稳定的一组，并按官方文档截至 2026-04-09 的信息整理。

## 第一部分：速查列表

### 启动与会话

| 命令 | 一句话作用 |
| --- | --- |
| claude | 在当前目录启动交互式 Claude Code 会话。 |
| claude "query" | 启动会话时顺手提交第一条任务。 |
| claude -p "query" | 单次执行，输出结果后直接退出。 |
| claude -c | 继续当前目录最近一次会话。 |
| claude -r "<session>" "query" | 恢复指定会话，并继续给它下任务。 |
| claude update | 把 Claude Code 更新到最新版本。 |
| claude --add-dir ../path | 临时把额外目录也纳入可读写范围。 |
| claude -w <name> | 在隔离的 git worktree 里启动会话。 |

## 第二部分：逐条详解

### 启动与会话

#### `claude`
- **功能**：进入交互模式，Claude 会读取当前项目上下文并在终端里持续对话。
- **常见场景**：刚进入一个仓库，准备让它先读代码、解释结构、再逐步改代码。
- **示例**：
```bash
claude
```
- **注意**：首次使用通常需要登录；命令应在项目根目录或子目录里执行。

#### `claude "query"`
- **功能**：创建交互式会话，并把引号里的内容作为首条消息发送，省掉先打开再输入一步。
- **常见场景**：你已经很明确知道第一件要做什么，比如“解释这个项目”或“定位登录 bug”。
- **示例**：
```bash
claude "explain this project"
```

#### `claude -p "query"`
- **功能**：以非交互模式运行一次任务，适合脚本、终端流水线或只要结果不要持续会话的场景。
- **常见场景**：你只想做一次性分析，比如检查某个函数、生成简短总结、配合 shell 脚本调用。
- **示例**：
```bash
claude -p "summarize the last commit"
```
- **注意**：常用于自动化；结束后不会像交互模式那样停留在会话里。

#### `claude -c`
- **功能**：直接接上最近一次对话，不用重新打开会话列表找上下文。
- **常见场景**：昨天已经分析到一半，今天想在同一个仓库里接着做。
- **示例**：
```bash
claude -c
```

#### `claude -r "<session>" "query"`
- **功能**：按会话 ID 或会话名恢复到特定历史会话，适合并行维护多个任务线。
- **常见场景**：同一个仓库里有“重构”“修 bug”“写测试”三条不同会话，需要精准接回某一条。
- **示例**：
```bash
claude -r "auth-refactor" "finish the middleware cleanup"
```

#### `claude update`
- **功能**：执行 CLI 更新，拿到新功能、命令变更和修复。
- **常见场景**：发现文档里的命令本地没有，或者遇到已知 bug，先更新版本最省事。
- **示例**：
```bash
claude update
```
- **注意**：不同安装方式的更新体验略有差异；原生安装通常更新更顺滑。

#### `claude --add-dir ../path`
- **功能**：给当前会话追加工作目录，让 Claude 同时访问多个相邻仓库或共享目录。
- **常见场景**：前端和后端分成两个目录，但你想让 Claude 一次跨目录联调。
- **示例**：
```bash
claude --add-dir ../frontend ../shared-lib
```
- **注意**：这是额外文件访问授权，不等于自动继承那些目录下的全部 .claude 配置。

#### `claude -w <name>`
- **功能**：为当前任务创建独立 worktree，避免把实验性改动直接弄脏主工作区。
- **常见场景**：并行做两个功能，或者让 Claude 大改代码时，你想先隔离环境再动手。
- **示例**：
```bash
claude -w feature-auth
```
- **注意**：适合中大型改动；本质是在 git worktree 上开一条安全支线。

### 会话内高频命令

#### `/help`
- **功能**：列出当前环境下可见的命令，帮助你快速确认某条命令是否可用。
- **常见场景**：忘了命令名，或者想先看本机当前版本支持哪些功能。
- **示例**：
```bash
/help
```

#### `/resume [session]`
- **功能**：不退出当前 Claude Code，也能直接打开会话选择器或指定某个会话继续。
- **常见场景**：你已经在 Claude 里，但想从当前会话切到另一条历史任务线。
- **示例**：
```bash
/resume auth-refactor
```

#### `/clear`
- **功能**：把当前会话历史清掉，释放上下文容量，避免旧指令继续干扰后续判断。
- **常见场景**：任务已经变了，或者上下文太乱，继续聊下去只会让回答越来越偏。
- **示例**：
```bash
/clear
```
- **注意**：它解决的是上下文污染，不会撤销已经改过的本地文件。

#### `/compact [instructions]`
- **功能**：把长对话浓缩成更短摘要，减少上下文占用，同时尽量保留当前任务所需信息。
- **常见场景**：长会话快吃满上下文，但你还想继续沿着当前任务往下做。
- **示例**：
```bash
/compact keep the current refactor plan and open TODOs
```
- **注意**：比 /clear 温和；它不是彻底重置，而是缩写历史。

#### `/init`
- **功能**：引导你为项目建立 Claude 的长期工作说明，比如代码风格、目录约定、测试方式和禁区。
- **常见场景**：你希望 Claude 每次进仓库都先按团队规则做事，而不是每次重新解释一遍。
- **示例**：
```bash
/init
```

#### `/config`
- **功能**：进入设置面板，统一修改会影响日常交互体验的参数。
- **常见场景**：想切主题、切输出风格、改编辑模式，或检查当前配置来源。
- **示例**：
```bash
/config
```

#### `/memory`
- **功能**：查看或编辑 CLAUDE.md 记忆文件，控制自动记忆是否启用，以及当前保存了哪些长期上下文。
- **常见场景**：你想让 Claude 持续记住项目规范、个人偏好、固定流程。
- **示例**：
```bash
/memory
```

#### `/permissions`
- **功能**：配置 Bash、读写文件等工具的权限策略，减少频繁确认或限制危险动作。
- **常见场景**：你经常重复批准同类安全命令，或者想明确禁止某类高风险操作。
- **示例**：
```bash
/permissions
```
- **注意**：这是效率和安全的平衡点；放得太宽会增加误操作风险。

#### `/model [model]`
- **功能**：在不同模型之间切换，通常用于在成本、速度、推理强度之间做取舍。
- **常见场景**：简单问答用更快模型，复杂重构或审查时切到更强模型。
- **示例**：
```bash
/model sonnet
```

#### `/diff`
- **功能**：打开交互式 diff 视图，快速核对 Claude 刚刚改了什么。
- **常见场景**：准备提交前做最后验收，或想审一下 Claude 某一轮改动是否越界。
- **示例**：
```bash
/diff
```

### 诊断与成本

#### `/cost`
- **功能**：显示会话成本或使用情况，帮助你判断这条任务是不是在无效烧额度。
- **常见场景**：长会话、复杂任务、团队共享额度时，想知道当前消耗是否合理。
- **示例**：
```bash
/cost
```

#### `/doctor`
- **功能**：做安装与配置层面的体检，定位为什么某些命令、登录、IDE 连接或环境行为异常。
- **常见场景**：命令失灵、登录反复失败、工具权限异常时，先跑它能省很多排查时间。
- **示例**：
```bash
/doctor
```

### 扩展能力：MCP 与插件

#### `/mcp`
- **功能**：在会话内查看 MCP 服务状态、做 OAuth 认证、确认工具是否已经接通。
- **常见场景**：你已经配置过 MCP，但不确定为什么 GitHub、Sentry、数据库工具没出现。
- **示例**：
```bash
/mcp
```

#### `claude mcp list`
- **功能**：从命令行看当前机器或项目上已经登记了哪些 MCP server。
- **常见场景**：接手别人机器或团队仓库时，先摸清已经挂了哪些外部能力。
- **示例**：
```bash
claude mcp list
```

#### `claude mcp add --transport stdio <name> -- <command>`
- **功能**：把一个本地进程形式的 MCP server 注册给 Claude，用于数据库、脚本、内部工具等场景。
- **常见场景**：你有本地 Python/Node 工具，希望 Claude 通过 MCP 直接调用。
- **示例**：
```bash
claude mcp add --transport stdio db -- python server.py
```
- **注意**：双横线 -- 用来分隔 Claude 的参数和 MCP 服务自身的启动命令。

#### `claude mcp add --transport http <name> <url>`
- **功能**：把远端 MCP 端点接进来，让 Claude 获得 GitHub、Sentry、Asana 等外部系统能力。
- **常见场景**：你要让 Claude 直接读 PR、查线上错误、查工单或访问团队服务。
- **示例**：
```bash
claude mcp add --transport http github https://api.githubcopilot.com/mcp/
```

#### `claude plugin install <plugin>@<marketplace>`
- **功能**：通过插件增加现成能力，比如代码审查、额外技能、挂钩或集成配置。
- **常见场景**：你不想从零写技能，想直接装一个成熟插件来补能力。
- **示例**：
```bash
claude plugin install code-review@claude-plugins-official
```
